# Руководство по использованию (USAGE)

Этот репозиторий содержит набор **навыков (skills) Claude** для инвесторов и трейдеров на рынке акций США. Каждый навык упаковывает специализированные промпты, базы знаний и вспомогательные скрипты для анализа рынка, технического анализа, мониторинга экономического календаря и разработки торговых стратегий. Навыки работают как в веб-приложении Claude, так и в Claude Code.

> ⚠️ **Важно:** часть навыков требует платных API-подписок (FMP API и/или FINVIZ Elite, Alpaca). Требования по каждому навыку указаны в таблице ниже.

## Условные обозначения

- ✅ **Обязательно** — навык не работает без этого API.
- 🟡 **Опционально / Рекомендуется** — навык работает без API, но с ним быстрее или точнее.

## Навыки по категориям

### 1. Режим рынка (market-regime)

Навыки для оценки общего состояния рынка: широта, тренды, риск вершины, макрорежим.

#### Market Breadth Analyzer 🟢
**Описание:** Количественная оценка здоровья рыночной широты на основе публичных CSV-данных TraderMonty. API не требуется.
**Кейсы использования:**
- Ежедневная проверка: «Насколько здорова широта рынка сегодня?»
- Получить композитный скоринг широты и разбивку по компонентам перед открытием позиций.

#### Uptrend Analyzer 🟢
**Описание:** Анализ рыночной широты по данным Monty's Uptrend Ratio Dashboard для диагностики текущей рыночной среды. API не требуется.
**Кейсы использования:**
- «Какой процент акций в восходящем тренде?»
- Ежедневная диагностика среды для принятия решений о размере экспозиции.

#### Exposure Coach 🟢
**Описание:** Формирует одностраничное резюме «Рыночной позиции» — потолок чистой экспозиции, смещение growth/value, рекомендацию «можно открывать новые позиции» vs «приоритет на кэш», интегрируя сигналы из навыков широты, режима и потоков.
**Кейсы использования:**
- «Какой максимальный уровень экспозиции допустим сегодня?»
- Свести воедино сигналы breadth/regime/flow в одну рекомендацию.

#### Market Top Detector 🟡
**Описание:** Оценка вероятности рыночной вершины по Distribution Days (O'Neil), деградации лидирующих акций (Minervini) и защитной ротации секторов (Monty). Публичные CSV, API не требуется.
**Кейсы использования:**
- «Близок ли рынок к вершине?»
- Раннее предупреждение для сокращения рисков.

#### FTD Detector 🟡 (требует **FMP ✅**)
**Описание:** Обнаружение сигналов Follow-Through Day для подтверждения дна рынка по методологии Уильяма О'Нила. Дневные OHLCV QQQ/SPY через FMP.
**Кейсы использования:**
- «Был ли подтверждён разворот рынка после коррекции?»
- Сигнал к возврату на рынок после медвежьей фазы.

#### IBD Distribution Day Monitor 🟡 (требует **FMP ✅**)
**Описание:** Отслеживание дней распределения для QQQ/SPY (закрытие минимум −0.2% на повышенном объёме), подсчёт кластеров d5/d15/d25, классификация риска (NORMAL/CAUTION/HIGH/SEVERE) и сигналы по TQQQ/QQQ.
**Кейсы использования:**
- Ежедневный мониторинг давления продаж институционалов.
- «Сколько дней распределения накопилось за последние 25 сессий?»

#### Breadth Chart Analyst 🟡 (вход: **скриншот графика**)
**Описание:** Анализ графиков рыночной широты, в частности S&P 500 Breadth Index (на базе 200-дневной MA) и графика доли акций в восходящем тренде.
**Кейсы использования:**
- Загрузить скриншот графика широты и получить интерпретацию.

#### Sector Analyst 🟡 (вход: **скриншот графика**)
**Описание:** Анализ паттернов ротации секторов и позиционирования в рыночном цикле.
**Кейсы использования:**
- «Какие секторы лидируют / отстают сейчас?»
- Определить фазу цикла по графику ротации.

#### Macro Regime Detector 🔴 (рекомендуется **yfinance / CSV**)
**Описание:** Обнаружение структурных переходов макрорежима (горизонт 1–2 года) через анализ кросс-активных соотношений.
**Кейсы использования:**
- «Меняется ли долгосрочный макрорежим (risk-on / risk-off)?»
- Кросс-активный анализ (акции/облигации, золото, кривая доходности).

#### Market Environment Analysis 🟡 (требует **WebSearch ✅**, опц. график)
**Описание:** Комплексный анализ рыночной среды с отчётностью. Глобальные данные через веб-поиск.
**Кейсы использования:**
- Утренний макробрифинг по мировым рынкам.
- «Какова общая картина рынка на сегодня?»

#### Market News Analyst 🟡 (требует **WebSearch ✅**)
**Описание:** Анализ недавних рыночных новостей и их влияния на акции и сырьё. Применяет скоринг влияния: (Price Impact × Breadth) × Forward Significance.
**Кейсы использования:**
- «Что важного произошло на рынке за последние 10 дней?»
- Ранжирование событий по количественному влиянию.

#### US Market Bubble Detector 🟡 (вход: **данные пользователя**)
**Описание:** Оценка риска пузыря через количественный анализ по обновлённому фреймворку Minsky/Kindleberger v2.1.
**Кейсы использования:**
- «Насколько высок риск пузыря на рынке США?»

### 2. Ядро портфеля (core-portfolio)

Навыки для долгосрочного дивидендного и базового портфеля.

#### Portfolio Manager 🟡 (требует **Alpaca ✅**)
**Описание:** Комплексный анализ портфеля через интеграцию с Alpaca MCP Server: загрузка позиций, анализ аллокации, риск-метрик, диверсификации и генерация рекомендаций по ребалансировке.
**Кейсы использования:**
- «Проанализируй мой портфель и предложи ребалансировку.»
- Мониторинг распределения активов и концентрации рисков.

#### IB Portfolio Manager 🟡 (требует **Interactive Brokers ✅**)
**Описание:** IB-версия Portfolio Manager — тот же фреймворк анализа, но данные берутся из Interactive Brokers через `interactive-brokers-mcp` (со встроенным IB Gateway). Только анализ, ордера не отправляет (рекомендуется `IB_READ_ONLY_MODE=true`). Поддерживает мультиактивные/мультивалютные счета IBKR.
**Кейсы использования:**
- «Проанализируй мой портфель в Interactive Brokers и предложи ребалансировку.»
- Обзор аллокации и рисков для счёта IBKR (без исполнения ордеров).

#### Value Dividend Screener 🟡 (требует **FMP ✅**, FINVIZ 🟡)
**Описание:** Скрининг акций США по дивидендам с фильтрами стоимости (P/E < 20, P/B < 2), привлекательной доходности (≥3%) и стабильного роста (дивиденды/выручка/EPS растут 3 года).
**Кейсы использования:**
- «Найди недооценённые дивидендные акции с доходностью от 3%.»
- Формирование доходного портфеля.

#### Dividend Growth Pullback Screener 🟡 (требует **FMP ✅**, FINVIZ 🟡)
**Описание:** Поиск качественных акций роста дивидендов (≥12% годового роста, ≥1.5% доходности) на временных откатах (RSI ≤40).
**Кейсы использования:**
- «Какие дивидендные растущие акции сейчас перепроданы?»
- Точечная докупка качественных бумаг на коррекции.

#### Kanchi Dividend SOP 🟡 (рекомендуется **FMP**)
**Описание:** Превращает дивидендное инвестирование в стиле «Kanchi» в повторяемую операционную процедуру для акций США (5-шаговый скрининг и планирование входа на откате).
**Кейсы использования:**
- «Проведи скрининг по методике Kanchi и подготовь памятку по акции.»

#### Kanchi Dividend Review Monitor 🟢 (рекомендуется **FMP**)
**Описание:** Мониторинг дивидендного портфеля с принудительными триггерами пересмотра (T1–T5), переводящий аномалии в статусы OK/WARN/REVIEW без автопродаж.
**Кейсы использования:**
- Еженедельная проверка портфеля на аномалии (срез дивиденда, просадка и т.д.).
- Формирование очереди позиций на пересмотр.

#### Kanchi Dividend US Tax Accounting 🟡 (расчёт, API не требуется)
**Описание:** Рабочий процесс по налогам на дивиденды США и размещению активов по типам счетов для портфелей в стиле Kanchi.
**Кейсы использования:**
- «Какие дивиденды квалифицированные, а какие обычные?»
- Рекомендации по размещению бумаг (taxable / IRA и т.д.).

### 3. Свинг-возможности (swing-opportunity)

Навыки для поиска и планирования среднесрочных трендовых сделок.

#### VCP Screener 🟡 (требует **FMP ✅**)
**Описание:** Скрининг S&P 500 на паттерн сжатия волатильности (VCP) Марка Минервини.
**Кейсы использования:**
- «Найди акции с готовым VCP-сетапом.»
- Поиск кандидатов на брейкаут.

#### CANSLIM Screener 🟡 (требует **FMP ✅**)
**Описание:** Скрининг акций США по методологии роста CANSLIM Уильяма О'Нила.
**Кейсы использования:**
- «Отбери растущие акции по критериям CANSLIM.»
- Композитный скоринг кандидатов.

#### Theme Detector 🟡 (FMP 🟡, FINVIZ 🟡)
**Описание:** Обнаружение и анализ трендовых рыночных тем по секторам с оценкой «теплового» скоринга.
**Кейсы использования:**
- «Какие темы / нарративы сейчас в моде на рынке?»
- Поиск горячих секторов и связанных акций.

#### Breakout Trade Planner 🟡 (расчёт + шаблоны ордеров Alpaca/IB)
**Описание:** Генерация торговых планов брейкаута в стиле Минервини из вывода VCP-скринера: расчёт риска по worst-case, управление «теплом» портфеля, шаблоны ордеров под брокера (`--broker alpaca|ib|both`) — Alpaca-формат (stop-limit для предразмещения, limit для входа после подтверждения) и/или последовательность ног `place_order` для interactive-brokers MCP.
**Кейсы использования:**
- «Подготовь план входа для кандидата из VCP-скринера.»
- Готовый шаблон ордера для брокера.

#### Finviz Screener 🟢 (FINVIZ 🟡)
**Описание:** Построение и открытие URL-скринеров FinViz из запросов на естественном языке.
**Кейсы использования:**
- «Собери скринер FinViz: small-cap, выше 50-MA, объём растёт.»

### 4. Планирование сделок (trade-planning)

Навыки для точного планирования конкретной сделки.

#### Position Sizer 🟢 (расчёт, API не требуется)
**Описание:** Расчёт размера позиции на основе риска для длинных сделок (по стопу, по ATR, по Kelly), с учётом ограничений портфеля.
**Кейсы использования:**
- «Сколько акций купить при риске 1% на сделку?»
- Расчёт с ограничениями по сектору и максимальной доле позиции.

#### Technical Analyst 🟡 (вход: **скриншот графика**)
**Описание:** Анализ недельных ценовых графиков акций, индексов, криптовалют и форекс-пар.
**Кейсы использования:**
- Загрузить недельный график и получить оценку тренда, уровней, сетапа.

#### US Stock Analysis 🟡 (вход: **данные пользователя**)
**Описание:** Комплексный анализ акции США: фундаментал (метрики, качество бизнеса, оценка), технический анализ (индикаторы, паттерны, поддержка/сопротивление), сравнение акций и генерация инвестиционного отчёта.
**Кейсы использования:**
- «Сделай полный разбор акции AAPL.»
- Сравнение двух-трёх акций перед выбором.

### 5. Память о сделках (trade-memory)

Навыки для ведения журнала, разбора результатов и обучения на сделках.

#### Trader Memory Core 🟢 (FMP 🟡, для MAE/MFE)
**Описание:** Отслеживание инвестиционных тезисов на протяжении всего жизненного цикла — от идеи в скринере до закрытой позиции с постмортемом.
**Кейсы использования:**
- Регистрация идеи из скринера → ENTRY_READY → ACTIVE → CLOSED.
- «Покажи все активные тезисы по AAPL» и расписание пересмотров.
- Частичные продажи (trim), фиксация P&L, генерация записи в журнал.

#### Signal Postmortem 🟢 (расчёт, API не требуется)
**Описание:** Запись и анализ результатов сделок по сигналам из edge-пайплайна и других навыков.
**Кейсы использования:**
- «Разбери закрытую сделку: что сработало, что нет.»
- Систематический пост-анализ.

#### Trade Performance Coach 🟡 (beta, расчёт, API не требуется)
**Описание:** Разбор закрытых сделок, частичных выходов и месячных агрегатов на предмет соблюдения процесса, дисциплины риска, качества исполнения и поведенческих паттернов; формирует правила работы на следующую сессию.
**Кейсы использования:**
- «Где я нарушаю свой торговый процесс?»
- Поведенческие теги и заметки риск-менеджера.

#### Trade Hypothesis Ideator 🔴 (расчёт, API не требуется)
**Описание:** Генерация фальсифицируемых гипотез торговых стратегий из рыночных данных, торговых логов и заметок журнала с ранжированными карточками гипотез и опциональным экспортом strategy.yaml.
**Кейсы использования:**
- «Сформулируй проверяемые гипотезы на основе моего журнала.»

### 6. Исследование стратегий (strategy-research)

Навыки для разработки, бэктеста и «edge»-исследований.

#### Backtest Expert 🔴 (вход: **параметры пользователя**)
**Описание:** Экспертное сопровождение систематического бэктеста торговых стратегий.
**Кейсы использования:**
- «Помоги корректно протестировать мою стратегию (избежать look-ahead bias и т.д.).»
- Оценка метрик производительности.

#### Strategy Pivot Designer 🔴 (расчёт, API не требуется)
**Описание:** Обнаружение стагнации итераций бэктеста и генерация структурно иных предложений пивота стратегии, когда подбор параметров упёрся в локальный оптимум.
**Кейсы использования:**
- «Я застрял на подгонке параметров — предложи принципиально иной подход.»

#### Scenario Analyzer 🔴 (требует **WebSearch ✅**)
**Описание:** Анализ 18-месячных сценариев из новостных заголовков через агента scenario-analyst со вторым мнением strategy-reviewer; первичный/вторичный/третичный анализ влияния и подбор акций.
**Кейсы использования:**
- «Какие последствия у этого заголовка на горизонте 18 месяцев?»
- Сценарный подбор бенефициаров и проигравших.

#### Edge Candidate Agent 🔴 (FMP 🟡)
**Описание:** Генерация и приоритизация исследовательских тикетов по long-side edge для акций США из EOD-наблюдений, экспорт спецификаций кандидатов для Phase I пайплайна стратегий.
**Кейсы использования:**
- Старт edge-пайплайна: market_summary.json + anomalies.json + tickets/.

#### Edge Hint Extractor 🟡 (расчёт, API не требуется)
**Описание:** Извлечение edge-подсказок из дневных рыночных наблюдений и реакций на новости (опционально с LLM-идеацией), вывод канонического hints.yaml.
**Кейсы использования:**
- Этап 2 edge-пайплайна — извлечение подсказок.

#### Edge Concept Synthesizer 🔴 (расчёт, API не требуется)
**Описание:** Абстрагирование тикетов детекторов и подсказок в переиспользуемые edge-концепции с тезисом, сигналами инвалидации и плейбуками стратегий.
**Кейсы использования:**
- Этап 3 edge-пайплайна — синтез концепций.

#### Edge Strategy Designer 🔴 (расчёт, API не требуется)
**Описание:** Преобразование абстрактных edge-концепций в варианты черновиков стратегий и опционально экспортируемые тикеты YAML.
**Кейсы использования:**
- Этап 4 edge-пайплайна — дизайн стратегий.

#### Edge Strategy Reviewer 🔴 (расчёт, API не требуется)
**Описание:** Критический разбор черновиков стратегий на правдоподобность edge, риск переобучения, адекватность размера выборки и реалистичность исполнения (PASS/REVISE/REJECT).
**Кейсы использования:**
- Этап 5 edge-пайплайна — рецензия стратегий.

#### Edge Signal Aggregator 🔴 (расчёт, API не требуется)
**Описание:** Агрегация и ранжирование сигналов из нескольких edge-навыков (edge-candidate-agent, theme-detector, sector-analyst, institutional-flow-tracker) в приоритизированный дашборд уверенности с взвешенным скорингом, дедупликацией и разрешением противоречий.
**Кейсы использования:**
- «Сведи все edge-сигналы в один рейтинг уверенности.»

#### Edge Pipeline Orchestrator 🔴 (расчёт, API не требуется)
**Описание:** Оркестрация всего edge-пайплайна от детекции кандидатов до дизайна, рецензии, ревизии и экспорта стратегий.
**Кейсы использования:**
- «Запусти полный edge-пайплайн автоматически с циклом обратной связи.»

#### Stanley Druckenmiller Investment 🔴 (расчёт, API не требуется)
**Описание:** Синтезатор стратегии Друкенмиллера — интегрирует вывод 8 апстрим-навыков (Market Breadth, Uptrend, Market Top, Macro Regime, FTD, VCP, Theme, CANSLIM) в единый скоринг уверенности (0–100).
**Кейсы использования:**
- «Свести всю картину рынка в единое решение в стиле Друкенмиллера.»

### 7. Продвинутые сателлитные стратегии (advanced-satellite)

Навыки для специализированных тактик (опционы, парный трейдинг, шорт, PEAD).

#### Options Strategy Advisor 🔴 (FMP 🟡)
**Описание:** Анализ и симуляция опционных стратегий: цена и греки по Блэку-Шоулзу, симуляция P&L, сравнение стратегий (covered call и др.).
**Кейсы использования:**
- «Рассчитай греки и P&L для covered call на AAPL.»
- Сравнение опционных стратегий.

#### Pair Trade Screener 🔴 (требует **FMP ✅**)
**Описание:** Инструмент статистического арбитража для поиска и анализа парных торговых возможностей (коинтеграция, z-score).
**Кейсы использования:**
- «Найди коинтегрированные пары в секторе Technology.»
- Анализ спреда конкретной пары (например, AAPL/MSFT).

#### Earnings Trade Analyzer 🟡 (требует **FMP ✅**)
**Описание:** Анализ акций после отчётности по 5-факторной системе скоринга (размер гэпа, предотчётный тренд, тренд объёма, позиция к MA200, позиция к MA50).
**Кейсы использования:**
- «Оцени реакции на недавнюю отчётность и отбери лучших кандидатов.»

#### PEAD Screener 🟡 (требует **FMP ✅**)
**Описание:** Скрининг акций с гэпом вверх после отчётности на паттерн PEAD (дрейф после объявления о прибыли).
**Кейсы использования:**
- «Найди акции с PEAD-паттерном (откат красной свечой → брейкаут).»
- Режим пайплайна от Earnings Trade Analyzer.

#### Parabolic Short Trade Planner 🔴 (требует **FMP ✅**, Alpaca 🟡)
**Описание:** Скрининг акций США на паттерны параболического истощения, генерация условных предрыночных шорт-планов и оценка срабатывания внутридневных триггеров по live 5-минутным барам.
**Кейсы использования:**
- «Найди параболические акции для шорта и подготовь предрыночный план.»
- Внутридневной мониторинг триггеров (ORL break, первая красная 5-мин свеча, VWAP fail).

#### Institutional Flow Tracker 🟡 (требует **FMP ✅**)
**Описание:** Отслеживание изменений институциональной собственности и потоков портфелей по данным 13F.
**Кейсы использования:**
- «Что покупали/продавали крупные фонды в прошлом квартале?»

## Готовые мультискилловые сценарии (workflows)

Канонические сценарии описаны в `workflows/*.yaml`. Это проверенные цепочки навыков для основного пользователя «Core + Satellite».

| Сценарий | Периодичность | Навыки |
|----------|---------------|--------|
| **market-regime-daily** | ежедневно | market-breadth-analyzer, uptrend-analyzer, exposure-coach (+ market-top-detector, macro-regime-detector) |
| **core-portfolio-weekly** | еженедельно | ib-portfolio-manager, trader-memory-core (+ value-dividend-screener, kanchi-*) |
| **swing-opportunity-daily** | ежедневно | vcp-screener, technical-analyst, position-sizer, trader-memory-core (+ theme-detector, breakout-trade-planner) |
| **trade-memory-loop** | по каждой закрытой сделке | trader-memory-core, signal-postmortem, trade-performance-coach |
| **monthly-performance-review** | ежемесячно | trader-memory-core, signal-postmortem, trade-performance-coach, dual-axis-skill-reviewer |

### Примеры прозаических цепочек

**Ежедневный мониторинг рынка:**
1. Economic Calendar Fetcher → события дня
2. Earnings Calendar → кто отчитывается
3. Market News Analyst → обзор ночных новостей
4. Breadth Chart Analyst → оценка здоровья рынка

**Торговля на отчётности (Earnings Momentum):**
1. Earnings Trade Analyzer → скоринг реакций (5 факторов)
2. PEAD Screener (режим B) → паттерны откат → брейкаут
3. Technical Analyst → подтверждение по недельному графику
4. Мониторинг входов со стопом и целями 2R

**Статистический арбитраж:**
1. Pair Trade Screener → коинтегрированные пары
2. Technical Analyst → подтверждение сетапов по обеим ногам
3. Мониторинг z-score и схождения спреда

**Edge-исследование (end-to-end):**
1. edge-candidate-agent → tickets/ + anomalies.json
2. edge-hint-extractor → hints.yaml
3. edge-concept-synthesizer → edge_concepts.yaml
4. edge-strategy-designer → strategy_drafts/*.yaml
5. edge-strategy-reviewer → review.yaml (PASS/REVISE/REJECT)
6. [REVISE] → ревизия → повторная рецензия (макс. 2 цикла)
7. [PASS] → экспорт стратегии
- **Оркестрованный режим:** edge-pipeline-orchestrator запускает все этапы автоматически.

**Параболический шорт (3 фазы):**
1. `screen_parabolic.py` → дневной watchlist (5-факторный скоринг → грейды A/B/C/D)
2. Промоут кандидатов A/B
3. `generate_pre_market_plan.py` → 3 плана триггеров на кандидата
4. Подтверждение блокирующих ручных условий у брокера (HTB locate и т.д.)
5. `monitor_intraday_trigger.py` → отслеживание FSM по 5-мин барам

## Настройка API-ключей

### Financial Modeling Prep (FMP)
```bash
export FMP_API_KEY=your_key_here
# либо через аргумент скрипта:
python3 economic-calendar-fetcher/scripts/get_economic_calendar.py --api-key YOUR_KEY
```
Регистрация: https://site.financialmodelingprep.com/developer/docs

### FINVIZ Elite
```bash
export FINVIZ_API_KEY=your_key_here
```
Регистрация: https://elite.finviz.com/

### Alpaca Trading
```bash
export ALPACA_API_KEY="your_api_key_id"
export ALPACA_SECRET_KEY="your_secret_key"
export ALPACA_PAPER="true"   # или "false" для реальной торговли
```
Регистрация: https://alpaca.markets/
Настройка MCP-сервера: см. `portfolio-manager/references/alpaca-mcp-setup.md`

### Interactive Brokers (IB Portfolio Manager)
MCP-сервер `interactive-brokers` через `npx` (Node.js 18+, IB Gateway в комплекте), запуск в режиме только-чтение для анализа:
```json
{
  "mcpServers": {
    "interactive-brokers": {
      "command": "npx",
      "args": ["-y", "interactive-brokers-mcp"],
      "env": { "IB_PAPER_TRADING": "true", "IB_READ_ONLY_MODE": "true" }
    }
  }
}
```
Регистрация: https://www.interactivebrokers.com/
Настройка MCP-сервера: см. `ib-portfolio-manager/references/ib-mcp-setup.md`
