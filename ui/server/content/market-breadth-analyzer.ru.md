# Market Breadth Analyzer — здоровье ширины рынка по CSV TraderMonty

## Кратко

Считает **композитный балл здоровья рынка 0–100** из 6 компонентов на базе публичного CSV TraderMonty. Измеряет, насколько **широко** рынок участвует в движении: рост на узком наборе лидеров и рост «всем фронтом» дают разный балл. Данные — индекс ширины TraderMonty (доля акций в аптренде, 8MA/200MA), API-ключ **не нужен**.

> Направление шкалы: **100 = максимальное здоровье** (широкое участие), 0 = критическая слабость. Это индикатор здоровья, а не риска — высокий балл хорош для лонга.

## Зачем в плане

Один из трёх входов **дневного гейта рынка** (воркфлоу `market-regime-daily`, Шаг 1). Балл и его JSON (`composite.composite_score`) уходят в `exposure-coach`, который сводит breadth + regime + flow в итог гейта `allow / restrict / cash-priority`. Сам по себе ответ на вопрос «рост broad-based или это иллюзия пары мегакапов?»: чем уже участие, тем раньше система переключается в защиту. Также формирует подсказку по уровню лонг-экспозиции и трендовый ярлык (improving/deteriorating/stable) для недельной сводки.

## Терминология

- **Breadth / ширина рынка** — какая доля акций реально растёт, а не только индекс. Узкая ширина = индекс тянут единицы.
- **Advance-decline** — баланс растущих и падающих бумаг; здесь выражен через долю акций в аптренде (индекс ширины 0–1).
- **8MA / 200MA** — сглаживания индекса ширины: 8-дневное (быстрое, тактика) и 200-дневное (медленное, режим).
- **Pink Zone** — «медвежья зона» из исходного дашборда: 200MA падает И 8MA ниже 200MA.

## Как работает

Вход — два CSV: detail (~2500 строк с 2016, колонки `Breadth_Index_8MA`, `Breadth_Index_200MA`, `Breadth_200MA_Trend`, `Bearish_Signal`, `Is_Peak`, `Is_Trough`, `Is_Trough_8MA_Below_04`, `S&P500_Price`) и summary (8 агрегатов: средние пики/впадины). При данных старше 5 дней — предупреждение.

### 6 компонентов и веса

| # | Компонент | Вес | Ключевой сигнал |
| - | --------- | --- | --------------- |
| 1 | Breadth Level & Trend | **25%** | уровень 8MA + направление 200MA + модификатор за 5-дн. направление 8MA |
| 2 | 8MA vs 200MA Crossover | **20%** | разрыв (gap) 8MA−200MA + направление 8MA |
| 3 | Peak/Trough Cycle | **20%** | позиция в цикле ширины относительно последнего пика/впадины |
| 4 | Bearish Signal | **15%** | бэктест-флаг + Pink Zone |
| 5 | Historical Percentile | **10%** | перцентиль текущего 8MA во всей истории |
| 6 | S&P 500 Divergence | **10%** | расхождение цены и ширины (окна 60д + 20д) |

Композит = взвешенная сумма. Если у компонента нет данных — его вес **пропорционально перераспределяется** между остальными.

### Пороги по компонентам (из кода)

- **К1 уровень 8MA → балл:** ≥0.70→95, ≥0.60→80, ≥0.50→65, ≥0.40→50, ≥0.30→35, ≥0.20→20, иначе 5. Тренд 200MA: +1→80, −1→20. Итог = 0.70·уровень + 0.30·тренд + модификатор (падение от высокого уровня −10; падение у дна +5; ранний рост +5).
- **К2 gap = 8MA−200MA → балл:** ≥+0.15→95, ≥+0.10→80, ≥+0.05→65, ≥0→50, ≥−0.05→35, ≥−0.10→20, иначе 5. Модификатор: gap<0 и 8MA растёт +10 (восстановление); gap>0 и 8MA падает −10.
- **К3 цикл:** окно 120 дней. После TROUGH (≤20дн и рост→85; 21–60→75; >60→65; без роста→30/35/40). После PEAK (≤20дн без роста→20; 21–60→15; >60→10; с ростом→50–60). Экстремальная впадина (`Is_Trough_8MA_Below_04`) +10. Нет маркера→50.
- **К4 база:** сигнал=False+аптренд→85; False+даунтренд→50; True+аптренд→30; True+даунтренд→10. Поправки: сигнал при 8MA>0.50 +15, при 8MA<0.25 −5; Pink Zone без сигнала −10.
- **К5 перцентиль:** ≥80→90, ≥60→70, ≥40→50, ≥20→30, иначе 10. Поправка: близко к ср. пику (≥0.95·avg_peak) −10, близко к ср. впадине (≤1.05·avg_trough) +10.
- **К6 дивергенция:** композит = 60д·0.6 + 20д·0.4. По окну: оба растут→70; оба падают→30; S&P +>3% и ширина <−0.05→10 (опасное расхождение); S&P −>3% и ширина >+0.05→80 (бычье расхождение).

### Зоны здоровья (100 = здоров)

| Балл | Зона | Лонг-экспозиция |
| ---- | ---- | --------------- |
| 80–100 | Strong | 90–100% |
| 60–79 | Healthy | 75–90% |
| 40–59 | Neutral | 60–75% |
| 20–39 | Weakening | 40–60% |
| 0–19 | Critical | 25–40% |

История баллов хранится по дате данных (до 20 записей) — отсюда трендовый ярлык.

## Команда

```bash
python3 skills/market-breadth-analyzer/scripts/market_breadth_analyzer.py \
  --output-dir trading-data/market/
```

URL-ы CSV подставляются по умолчанию (`tradermonty.github.io/market-breadth-analysis/...`), ключ не требуется. Выдаёт `market_breadth_<timestamp>.json` + `.md` и историю `market_breadth_history.json` в `trading-data/market/`. Внутри: композит и зона, разбивка по 6 компонентам с эффективными весами, сильнейший/слабейший компонент, ключевые уровни (0.40 экстремальная слабость, 0.60 здоровый порог, ср. пик/впадина), предупреждение о свежести данных.
