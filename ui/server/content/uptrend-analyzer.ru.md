# Uptrend Analyzer — ширина рынка по дашборду Uptrend Ratio

## Кратко

Диагностирует здоровье рынка через **Monty Uptrend Ratio Dashboard** (~2800 акций США в 11 секторах). Композитный балл **0–100** из 5 компонентов: чем выше — тем шире и здоровее аптренд. В отличие от `market-top-detector` (платный риск-скорер), работает на бесплатном GitHub-CSV, ключ **не нужен**.

> Шкала: **100 = широкий здоровый рост**, 0 = медвежий рынок. Высокий балл поддерживает лонг-экспозицию.

## Зачем в плане

Второй breadth-вход **дневного гейта рынка** (воркфлоу `market-regime-daily`, Шаг 1) рядом с `market-breadth-analyzer`. JSON (`composite.composite_score`, либо `uptrend_pct`) уходит в `exposure-coach`, где вместе с regime/flow формирует итог гейта `allow / restrict / cash-priority`. Отвечает на вопрос «насколько широк фронт участников» по секторам и даёт ярлык участия broad/narrow, влияющий на потолок экспозиции.

## Терминология

- **Uptrend ratio** — доля акций в аптренде (0–1); по дашборду пороги Monty: overbought 0.37, oversold 0.097.
- **Sector participation** — сколько из 11 секторов в аптренде и насколько ровно (spread между сильнейшим и слабейшим).
- **Rotation** — кто ведёт: циклика (risk-on) или защита (risk-off); сырьё выше обеих групп → late-cycle.

## Как работает

Вход — два CSV: timeseries (`all` + 11 секторов, ratio/slope/trend/10MA, с 2023/08) и sector_summary (последний срез); при отсутствии summary он достраивается из timeseries.

### 5 компонентов и веса

| # | Компонент | Вес | Ключевой сигнал |
| - | --------- | --- | --------------- |
| 1 | Market Breadth (Overall) | **30%** | уровень ratio + направление тренда |
| 2 | Sector Participation | **25%** | число аптренд-секторов + spread |
| 3 | Sector Rotation | **15%** | баланс циклики vs защиты |
| 4 | Momentum | **20%** | наклон (slope) + ускорение + ширина наклонов по секторам |
| 5 | Historical Context | **10%** | перцентиль текущего ratio в истории |

Композит = взвешенная сумма (без перераспределения весов; нет данных → компонент берёт нейтраль).

### Пороги по компонентам (из кода)

- **К1 ratio→балл (линейная интерполяция):** ≥0.50→90–100; 0.37–0.50→70–89; 0.25–0.37→40–69; 0.097–0.25→10–39; <0.097→0–9. Поправка тренда: up & slope>0 → +5, down & slope<0 → −5.
- **К2 = count·0.60 + spread·0.40.** Count (из 11): 10–11→100, 8–9→80, 6–7→60, 4–5→40, 2–3→20, 0–1→0. Spread (max−min ratio): <0.15→100, 0.15–0.25→~80, 0.25–0.35→~60, 0.35–0.45→~30, >0.45→0.
- **К3 = cyclical_avg − defensive_avg:** >+0.15→90–100; +0.05…+0.15→70–89; −0.05…+0.05→45–69; −0.15…−0.05→20–44; <−0.15→0–19. Циклика: Technology, Consumer Cyclical, Communication Services, Financial, Industrials. Защита: Utilities, Consumer Defensive, Healthcare, Real Estate. Сырьё (Energy, Basic Materials) выше обеих групп → флаг late-cycle + штраф.
- **К4 = slope·0.50 + acceleration·0.30 + sector_breadth·0.20.** Наклон сглажен EMA(3); ускорение — окно 10v10 (fallback 5v5); ширина наклонов — доля секторов с положительным slope.
- **К5:** балл = перцентиль текущего ratio в истории (прямое отображение); рядом min/max/median, 30д/90д средние и оценка доверия (High/Medium/Low по объёму выборки, охвату режимов и свежести).

### Предупреждения (штрафы после композита)

| Warning | Условие | Штраф |
| ------- | ------- | ----- |
| Late Cycle | сырьё > и циклики, и защиты | −5 |
| High Spread | spread секторов > 40 п.п. | −3 |
| Divergence | расхождение внутри групп | −3 |

Штрафы суммируются (макс −10), при ≥2 активных — скидка +1.

### Зоны (выше = здоровее)

| Балл | Зона | Экспозиция |
| ---- | ---- | ---------- |
| 80–100 | Strong Bull | Full (100%) |
| 60–79 | Bull | Normal (80–100%) |
| 40–59 | Neutral | Reduced (60–80%) |
| 20–39 | Cautious | Defensive (30–60%) |
| 0–19 | Bear | Preservation (0–30%) |

Каждая зона дробится на 7 подзон (Bull-Upper/Lower и т.д.) для тонкой настройки.

## Команда

```bash
python3 skills/uptrend-analyzer/scripts/uptrend_analyzer.py \
  --output-dir trading-data/market/
```

URL-ы CSV (`raw.githubusercontent.com/tradermonty/uptrend-dashboard/...`) зашиты, ключ не нужен. Выдаёт `uptrend_analysis_<timestamp>.json` + `.md` в `trading-data/market/`. Внутри: композит, зона и 7-уровневый детайл, разбивка по 5 компонентам, активные предупреждения и штраф, сильнейший/слабейший компонент, секторный heatmap.
