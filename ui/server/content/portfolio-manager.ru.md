# Portfolio Manager — работа с брокером Alpaca через MCP

## Кратко

Подключается к брокеру **Alpaca через MCP** и работает с реальным (paper) счётом: тянет позиции и equity, считает аллокацию, риск-метрики и диверсификацию, даёт рекомендации по ребалансу и **исполняет bracket-ордера**. В отличие от калькуляторов плана, это единственный скил, который реально касается счёта. **Требует** настроенного Alpaca MCP Server (см. `references/alpaca-mcp-setup.md`).

> Анализ строится на живых данных счёта (`get_account_info`, `get_positions`, `get_portfolio_history`), а не на введённых вручную цифрах. Капитал плана — $150k на Alpaca paper.

## Зачем в плане

Две точки исполнения:

- **Шаг 2.3** — поставить **bracket-ордер по плану** на завтрашний вход (вход + стоп + цель одним ордером) по уровням из `breakout-trade-planner` (Шаг 5.4).
- **Шаг 6.5** — поставить **sell-short bracket** по шорт-кандидату (шорт-ветка включается только при гейте `restrict`/`cash-priority`; правила шорта — из Шага 6.4).

Также используется для обзора портфеля: аллокация, концентрация, риск, ребаланс.

## Как работает

### Источник данных (Alpaca MCP)

Скил вызывает MCP-инструменты Alpaca: `get_account_info` (equity, кэш, buying power), `get_positions` (тикер, количество, средняя цена, рыночная стоимость, нереализованный P&L, доля в портфеле), опционально `get_portfolio_history` (история equity, просадка). Если MCP не подключён — скил сообщает и даёт инструкции по настройке, либо предлагает ручной ввод позиций как фолбэк.

### Что считает

- **Аллокация** — по классам активов, секторам, капитализации, географии; сравнение с целевой моделью под риск-профиль.
- **Диверсификация** — концентрация позиций (флаг, если одна > 10–15%), концентрация сектора (флаг, если > 30–40%), число позиций (оптимум 15–30), корреляции (поиск избыточных пар).
- **Риск** — оценочная бета портфеля, просадка, доля в высоковолатильных бумагах, tail-risk; итоговый риск-скор.
- **Ребаланс** — что подрезать (overweight, сломанный тезис, концентрация), что добрать, как развернуть кэш; приоритеты (immediate / high / medium / low).

### Исполнение ордеров

Ордера ставятся как **bracket** (вход + take-profit + stop-loss одной заявкой) — стоп выставляется одновременно со входом, не «потом». Для шорта — `sell-short` bracket (стоп выше входа, цели ниже).

> **ВАЖНО — запуск интерактивный.** В плане этот скил вызывается через `claude "..."` (НЕ через `claude -p`): автопилот сам ордера не отправляет. Ты видишь сформированный ордер глазами и подтверждаешь его вручную до отправки брокеру. Это сознательный barrier между планом и реальными деньгами.

### Плановый вызов (Шаг 2.3 — поставить bracket по плану)

```text
claude "Открой skill portfolio-manager (Alpaca paper). Поставь bracket-ордер
по плану: <ТИКЕР>, buy-stop $<вход>, стоп $<стоп>, тейк $<цель>, <N> акций.
Покажи ордер до отправки и жди моего подтверждения."
```

Для Шага 6.5 — тот же интерактивный вызов, но в формулировке указать **sell-short bracket** (стоп выше входа), уровни взять из шорт-плана.

## Команда

Интерактивный режим, без `-p` (ордера подтверждаются глазами):

```bash
# Шаг 2.3 — bracket-ордер на вход по плану (вход + стоп + цель)
claude "Открой skill portfolio-manager (Alpaca paper). Поставь bracket-ордер по плану: \
<ТИКЕР>, buy-stop \$<вход>, стоп \$<стоп>, тейк \$<цель>, <N> акций. \
Покажи ордер до отправки и жди подтверждения."

# Шаг 6.5 — sell-short bracket по шорт-кандидату (стоп ВЫШЕ входа)
claude "Открой skill portfolio-manager (Alpaca paper). Поставь sell-short bracket: \
<ТИКЕР>, шорт по \$<вход>, стоп \$<стоп выше входа>, тейк \$<цель ниже>, <N> акций. \
Покажи ордер до отправки и жди подтверждения."

# Обзор портфеля (только чтение)
claude "Открой skill portfolio-manager и сделай обзор портфеля: аллокация, риск, ребаланс."
```

Скил не пишет в `trading-data/` — результат исполнения (фактическая цена/дата входа) переносится в журнал отдельно (тезис в `trader-memory-core`).
