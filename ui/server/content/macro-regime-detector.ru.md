# Macro Regime Detector — структурный макрорежим рынка

## Кратко

Определяет **структурный макрорежим** и его смены на горизонте **1–2 года** по кросс-ассетным ratio месячной частоты. Композит **0–100 «сила сигнала перехода»** из шести компонентов плюс классификация в один из **пяти режимов**. API-ключ не нужен — берёт 600 дней истории по 9 ETF + доходностям казначеек через общий слой TradingView (`--api-key` / `$FMP_API_KEY` принимаются, но игнорируются).

> Это **фон**, а не сигнал входа: горизонт 1–2 года против тактических 2–8 недель у `market-top-detector` и снимка у `market-breadth-analyzer`. Задаёт направление: какой режим благоприятен для лонга, какой — для шортов и защиты.

## Зачем в плане

В **субботнем недельном блоке (Шаг 8)** даёт структурный контекст для лонг/шорт-решений на неделю. Меняется медленно — **раз в 2–4 недели прогона достаточно**, ежедневно пересчитывать незачем. Помогает понять, согласуется ли тактика (вершина/дно/ширина) с долгосрочной ротацией.

## Терминология

- **RSP/SPY** — равновзвешенный S&P против капвзвешенного: падает = концентрация в мегакапах, растёт = расширение участия.
- **10Y–2Y spread** — наклон кривой доходности (инверсия < 0; крутая ≥ 1.5%).
- **HYG/LQD** — высокодоходные облигации против инвестгрейда = аппетит к кредитному риску.
- **IWM/SPY** — малые капы против рынка (size-фактор).
- **SPY/TLT + корреляция** — акции против длинных облигаций; знак корреляции = режим хеджирования.
- **XLY/XLP** — циклика против защиты (risk-on / risk-off).

## Как работает

Композит = взвешенная сумма шести компонентов (каждый 0–100):

| # | Компонент | Ratio/данные | Вес | Что детектирует |
| - | --------- | ------------ | --- | --------------- |
| 1 | Market Concentration | RSP/SPY | **25%** | Концентрация в мегакапах против расширения |
| 2 | Yield Curve | 10Y–2Y | **20%** | Переходы цикла ставок |
| 3 | Credit Conditions | HYG/LQD | **15%** | Кредитный цикл, аппетит к риску |
| 4 | Size Factor | IWM/SPY | **15%** | Ротация малые ↔ крупные капы |
| 5 | Equity-Bond | SPY/TLT + корр. | **15%** | Режим связи акции-облигации |
| 6 | Sector Rotation | XLY/XLP | **10%** | Циклика против защиты |

### Балл компонента (0–100)

Каждый компонент даунсемплится в месячные точки (нужно ≥12 месяцев) и оценивается единым движком `score_transition_signal` в три слоя:

1. **Кроссовер 6M/12M SMA (0–40):** свежий (≤2 мес.) → 40, недавний (≤5) → 30, старый → 20; сходящиеся MA (|gap| < 1%) → до 25.
2. **Сдвиг импульса (0–30):** разворот короткого ROC против длинного тренда = раннее предупреждение; сильный импульс по тренду (>3%) → +10.
3. **Подтверждение (0–30):** +10 за каждый согласованный сигнал (кроссовер есть / короткий ROC подтверждает / зазор SMA расширяется).

Equity-Bond добавляет бонус за смену знака корреляции акции-облигации (до +20). Зоны композита: 0–20 Stable → 21–40 Early Signal → 41–60 Transition Zone → 61–80 Active Transition → 81–100 Confirmed Transition.

### Пять режимов

Классификатор отдельно от композита считает «очки» за каждую гипотезу по направлениям компонентов и выбирает лучшую:

- **Concentration** — RSP/SPY вниз, лидерство мегакапов, узкий рынок, кредит стабилен. Фокус на крупных техах, недовес малых/value.
- **Broadening** — RSP/SPY вверх, малые капы догоняют, кредит мягкий/стабилен. Добавлять малые/средние капы, value, цикличку.
- **Contraction** — кредит ужесточается, защитная ротация, equity-bond risk-off, поздний цикл. Поднять кэш, защита (staples/healthcare), управление дюрацией.
- **Inflationary** — **положительная** корреляция акции-облигации, обычный хедж ломается. Реальные активы, сырьё, энергия, короткая дюрация, TIPS.
- **Transitional** — несколько сигналов, но без ясного паттерна (или ничья при низком композите). Диверсификация, плавная перебалансировка, мониторинг.

Уверенность (high/moderate/low/very_low) ограничивается полнотой данных: при ≤3 доступных компонентах — не выше very_low, при ≤4 — не выше low. Дополнительно считается вероятность того, что переход уже идёт (от «<20%» до «70–90%»), с направлением «из режима → в режим».

## Команда

```bash
python3 skills/macro-regime-detector/scripts/macro_regime_detector.py \
  --output-dir trading-data/market/
```

- `--days N` — глубина истории (по умолчанию 600).
- `--api-key` — только для обратной совместимости, игнорируется.

Выдаёт JSON + Markdown в `trading-data/market/`: текущий режим и уверенность, дашборд сигналов перехода, детали по 6 компонентам, доказательная база режима и рекомендации по позиционированию портфеля.
