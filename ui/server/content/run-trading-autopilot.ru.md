# Автопилот — самодиспетчер слотов по расписанию

## Кратко

`scripts/run_trading_autopilot.py` — тонкая обёртка над слот-раннером. Запускается cron/launchd **каждые 15 минут**, сам смотрит на часы (CET), торговый календарь США и своё состояние, решает, какой слот сейчас уместен, и делегирует его `run_trading_schedule.py --slot ...`. Ничего нового не считает — это диспетчер времени.

> В Telegram автопилот шлёт **только важное** (сбой слота, исчерпание повторов, смена гейта экспозиции, серия падений интрадея). Успешные слоты сами присылают свои дайджесты — автопилот их не дублирует.

## Зачем в плане

Автопилот — то, что превращает ручной слот-раннер в фоновую систему. По будням он сам отрабатывает Шаг 1 (премаркет), Шаг 3 (интрадей-мониторинг), Шаг 4 (вечер); по выходным — Шаг 8 (суббота) и Шаг 9 (1-е воскресенье). Если ожидаемого Telegram нет — значит автопилот не отработал, и ты запускаешь нужный слот руками через `run_trading_schedule.py` (ручной фолбэк).

## Терминология

- **Слот** — один шаг дня, который автопилот делегирует слот-раннеру.
- **Окно** — диапазон локального времени, в котором слот вообще может запуститься (границы привязаны к US-сессии, пересчёт из ET под DST и короткие сессии).
- **Дедуп** — раз-в-день слоты не повторяются после успеха; провалившийся повторяется до `MAX_ATTEMPTS = 2`, пока окно открыто.
- **Гейт экспозиции** — `exposure_decision` за день; автопилот замечает его смену allow ↔ restrict/cash-priority и шлёт алерт.
- **Лок занят (exit 75)** — слот-раннер уже работает (например, ручной прогон): автопилот тихо откатывает попытку и повторит на следующем тике, без ошибки и Telegram.

## Как работает

### Выбор слота по времени (CET)

| Окно (CET) | Слот | Повторяемость |
| --- | --- | --- |
| 15:00–21:00, торговый день США | `premarket` | 1 раз/день; провал → 1 повтор (всего 2 попытки) |
| 15:30–22:00, торговый день США | `intraday` | каждые ~15 мин, мониторинг весь день |
| 22:15–24:00, торговый день США | `evening-prep` | 1 раз/день; провал → повтор до 2 попыток |
| суббота с 12:00 | `weekly` | 1 раз/неделю |
| 1-е воскресенье с 11:00 | `monthly` | 1 раз/месяц |
| вне окон / не торговый день | `none` | «шагов нет» (пишется в лог) |

Приоритет: monthly/weekly → (если торговый день) evening-prep → premarket → intraday. Интрадей — единственный повторяемый: новый запуск не раньше чем через `INTRADAY_INTERVAL_MIN ≈ 15` мин (с допуском 1 мин, иначе через тик цикл вырождался в 30-минутный).

### Интрадей-мониторинг

Дешёвый чек **без TradingView Desktop и без API-ключей** — котировки тянутся публичным путём. Сравнивает цены watchlist и открытых позиций с уровнями → сигналы **ОТКРОЙ** (триггер сработал), **ЗАКРОЙ**-типа (стоп / у-стопа / **+2R**). Каждый сигнал уходит в Telegram один раз за день (дедуп). Алерт о сбое — только после `INTRADAY_FAILURE_ALERT_STREAK = 3` падений подряд (сбрасывается на успехе).

### Идемпотентность и попытки

Перед запуском слота попытка фиксируется в состоянии — даже краш засчитывается в счётчик повторов. Раз-в-день слот со статусом `done` не перезапускается; со статусом `failed` и исчерпанными попытками — тоже (нужен ручной разбор). На `exit 75` (лок занят) попытка откатывается, слот остаётся диспетчеризуемым.

### Что уходит в Telegram

Только важное: слот упал (`rc≠0`) — с пометкой, повторит он или попытки исчерпаны; **3 падения интрадея подряд**; смена гейта экспозиции (`allow → restrict` и т.п.; деградированный fail-safe-гейт за смену не считается — это шум). Успешные дайджесты шлёт сам слот-раннер.

### Состояние и логи

- `trading-data/logs/autopilot/autopilot_<ts>.log` — подробный лог каждого прогона (решение, причина, вывод дочернего слота, состояние до/после), включая no-op; хранится 30 дней.
- `trading-data/logs/autopilot_state.json` — статусы слотов за день, история weekly/monthly, последнее решение гейта.
- `trading-data/logs/intraday_signals_state.json` — дедуп уже отправленных интрадей-сигналов.
- `trading-data/logs/autopilot.lock` — PID-лок против наложения прогонов автопилота.

### Флаги

| Флаг | Назначение |
| --- | --- |
| `--dry-run` | Решает и логирует, но состояние не меняет и ничего не шлёт; слот-раннер тоже в `--dry-run`. |
| `--now <ISO>` | Подменить «сейчас» (например `2026-06-10T16:05:00`) для тестов. |
| `--force-slot <slot>` | Обойти окна/дедуп и запустить конкретный слот немедленно. |
| `--no-telegram` | Подавить все Telegram-сообщения. |
| `--timeout <сек>` | Таймаут claude-воркфлоу, прокидывается в слот-раннер (по умолчанию 1800). |

## Команда

Ставится в cron каждые 15 минут (интрадею нужна эта частота, остальные слоты дедупятся сами):

```bash
*/15 * * * * cd <repo> && /usr/bin/python3 scripts/run_trading_autopilot.py >> trading-data/logs/autopilot_cron.log 2>&1
```

Ручная проверка (ничего не меняет и не шлёт):

```bash
# что бы сделал прямо сейчас
python3 scripts/run_trading_autopilot.py --dry-run

# что бы сделал в заданный момент
python3 scripts/run_trading_autopilot.py --now 2026-06-10T16:05:00 --dry-run

# форсировать интрадей-чек немедленно
python3 scripts/run_trading_autopilot.py --force-slot intraday
```

Артефакты слотов пишутся туда же, куда у слот-раннера (`trading-data/schedule|market|screeners|plans|journal/`); автопилот добавляет только свои логи/состояние в `trading-data/logs/`.
