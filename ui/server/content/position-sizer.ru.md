# Position Sizer — риск-базированный сайзинг лонга

## Кратко

Считает **количество акций** для лонг-сделки от заданного риска: сколько брать, чтобы при срабатывании стопа потерять ровно запланированную сумму. Базовая формула — `акции = риск$ / (вход − стоп)`. Поддерживает три метода (фиксированная доля, ATR, Kelly) и режет результат лимитами по позиции и сектору. Чистая калькуляция — ни сети, ни ключей, только стандартная библиотека Python.

> В плане дня это инструмент для **НЕ-VCP кандидата**: когда сделка пришла не из `vcp-screener` (а, например, из разбора тикера), и уровни вход/стоп взяты из `ticker-analysis`. Для VCP-кандидатов сайзинг уже встроен в `breakout-trade-planner`.

## Зачем в плане

**Шаг 5.4** (ручная ветка). Основной путь Шага 5.4 — `breakout-trade-planner` по VCP-выводу. Но если кандидат нестандартный — не из VCP, уровни из ручного разбора, — размер позиции считается этим скриптом: даёшь `--entry` и `--stop` (из `ticker-analysis`), получаешь количество акций под риск 1% с проверкой лимитов. Результат ложится в `trading-data/plans/` (`position_sizer_*`).

## Терминология

- **R / риск на сделку** — сумма, которой рискуешь, если цена дойдёт до стопа. В плане дня R = 1% капитала = **$1500** при $150k. R на акцию = `вход − стоп`.
- **Fixed Fractional** — основной метод: рискуешь фиксированным % капитала на сделку (по умолчанию 1%).
- **ATR (Average True Range)** — средний истинный диапазон, мера дневной волатильности. ATR-сайзинг ставит стоп на расстоянии `ATR × множитель` (по умолчанию ×2.0) — стоп шире на волатильных бумагах.
- **Kelly** — оптимальная доля капитала из исторической статистики (винрейт, средний выигрыш/проигрыш). На практике берётся **half-Kelly** (половина) — 75% роста при куда меньшем риске.

## Как работает

### Три метода

1. **Fixed Fractional** (основной): `риск$ = капитал × риск% / 100`; `акции = ⌊риск$ / (вход − стоп)⌋`. Стоп задаётся явно через `--stop`.
2. **ATR-based**: `дистанция_стопа = ATR × множитель`; `стоп = вход − дистанция`; дальше та же формула от `дистанция_стопа` как риска на акцию. Включается флагом `--atr` (+ `--atr-multiplier`, дефолт 2.0).
3. **Kelly**: `Kelly% = W − (1−W)/R`, где `W` = винрейт, `R` = средний_выигрыш / средний_проигрыш; берётся `half-Kelly = Kelly%/2`, отрицательное матожидание обнуляется. Без `--entry` — выдаёт только бюджет риска ($ и %); с `--entry`/`--stop` — переводит бюджет в акции.

### Лимиты портфеля (берётся минимум)

После расчёта по методу применяются ограничения, и **выигрывает самый строгий**:

- **`--max-position-pct`** — позиция ≤ N% капитала: `макс_акций = ⌊капитал × N% / 100 / вход⌋`. В плане — 25%.
- **`--max-sector-pct`** — сектор ≤ N% с учётом уже занятого (`--current-sector-exposure`): `остаток% = N% − занято%`, `макс_акций = ⌊остаток$ / вход⌋`. В плане — 30%.

Итог = `min(акции_по_риску, макс_по_позиции, макс_по_сектору)`, округление **вниз** до целого. В отчёт пишется, какой лимит связал (`binding_constraint`), фактический риск в $ и % капитала.

> Правила сайзинга (из скила): выживание важнее максимизации; дефолт 1%, без исключительной причины не выше 2%; всегда округляй вниз; совокупный heat портфеля ≤ 6–8%; никогда не full-Kelly.

### Профиль

`--profile` (или `$TRADING_PROFILE`, дефолтом — `trading-data/trading_profile.json`) подтягивает капитал, риск% и лимиты из общего JSON. Явные CLI-флаги всегда перебивают профиль. Профиль-`risk_pct` не блокирует Kelly-режим — конфликтует только явный `--risk-pct` с `--win-rate`.

## Команда

```bash
# Шаг 5.4 (НЕ-VCP) — сайзинг по уровням из разбора тикера; капитал/риск/лимиты из профиля
python3 skills/position-sizer/scripts/position_sizer.py --entry <вход> --stop <стоп>

# Явный риск и лимиты без профиля
python3 skills/position-sizer/scripts/position_sizer.py \
  --account-size 150000 --entry <вход> --stop <стоп> \
  --risk-pct 1.0 --max-position-pct 25 --max-sector-pct 30 --current-sector-exposure 22

# ATR-вариант стопа
python3 skills/position-sizer/scripts/position_sizer.py --entry <вход> --atr <ATR> --atr-multiplier 2.0
```

Выдаёт `position_sizer_<дата>_<время>.json` и `.md` в `trading-data/plans/`: метод, число акций, стоимость позиции, риск в $ и %, связавший лимит. `--output-dir` по умолчанию уже `trading-data/plans/`.
