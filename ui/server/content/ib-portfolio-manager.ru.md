# IB Portfolio Manager — анализ портфеля через Interactive Brokers MCP

## Кратко

IB-версия `portfolio-manager`: тот же фреймворк анализа портфеля, но источник данных — **Interactive Brokers через MCP** (`interactive-brokers-mcp`, со встроенным IB Gateway), а не Alpaca. Тянет позиции, балансы и ордера со счёта IBKR (paper или live), считает аллокацию, риск-метрики, диверсификацию и даёт рекомендации по ребалансу.

> **Только анализ, без исполнения.** В отличие от Alpaca-версии (которая ставит bracket-ордера на шагах плана 2.3 / 6.5), этот скил **не отправляет ордера**. Рекомендуется запускать MCP с `IB_READ_ONLY_MODE=true`, чтобы `place_order` был отключён на уровне сервера. Исполнение остаётся за пользователем у брокера.

## Когда использовать

Когда брокерский счёт — в **Interactive Brokers**, а не в Alpaca. Это on-demand обзор портфеля (аллокация / риск / концентрация / ребаланс), а **не** часть автопилота торгового расписания — автоматическое исполнение в плане по-прежнему идёт через Alpaca `portfolio-manager`.

## Как работает

### Источник данных (Interactive Brokers MCP)

Скил вызывает MCP-инструменты IB: `get_account_info` (net liquidation, кэш, buying power), `get_positions` (символ/conid, класс актива STK/OPT/FUT/CASH/BOND, количество, средняя цена, рыночная стоимость, нереализованный P&L), `get_live_orders` (рабочие заявки, чтобы не дублировать ребаланс), опционально `get_flex_query` для исторической динамики и реализованного P&L (нужен `IB_FLEX_TOKEN`). У IB нет единого `get_portfolio_history` как у Alpaca — история берётся из Flex-выгрузок. Если MCP не подключён — скил сообщает и даёт инструкции (`references/ib-mcp-setup.md`) либо предлагает ручной ввод позиций (CSV) как фолбэк.

### Что считает

- **Аллокация** — по классам активов, секторам, капитализации, географии и **валюте** (IBKR мультивалютный); сравнение с целевой моделью под риск-профиль.
- **Диверсификация** — концентрация позиций (флаг > 10–15%), сектора (флаг > 30–40%), число позиций (оптимум 15–30), корреляции.
- **Риск** — оценочная бета, просадка (если есть Flex-история), плечо/маржа, экспозиция опционов/фьючерсов (notional и delta-adjusted), tail-risk; итоговый риск-скор.
- **Ребаланс** — что подрезать, что добрать, как развернуть кэш; приоритеты (immediate / high / medium / low). Результат — **план для ручного исполнения**, не ордера.

### Настройка

MCP-сервер `interactive-brokers` ставится через `npx -y interactive-brokers-mcp` (Node.js 18+, IB Gateway в комплекте). Авторизация — браузерная (OAuth) или headless (`IB_USERNAME` / `IB_PASSWORD_AUTH`). Диагностика подключения:

```bash
python3 skills/ib-portfolio-manager/scripts/check_ib_connection.py
```

## Команда

```bash
# Обзор портфеля IBKR (только чтение)
claude "Открой skill ib-portfolio-manager и сделай обзор портфеля Interactive Brokers: \
аллокация, риск, концентрация, ребаланс."
```

Отчёт сохраняется в `reports/ib_portfolio_analysis_YYYY-MM-DD.md`. Скил не пишет в `trading-data/` и не отправляет ордера.
