# Market Top Detector — вероятность вершины рынка

## Кратко

Считает композит **0–100 «вероятность формирования вершины»** из шести компонентов, объединяя три проверенные методики: **O'Neil** (дни распределения), **Minervini** (деградация лидеров) и **Monty** (защитная ротация секторов). API-ключ не нужен — котировки/история S&P 500, QQQ, VIX и корзин ETF идут через общий слой TradingView (`--api-key` / `$FMP_API_KEY` принимаются, но игнорируются).

> «Оборонительный» индикатор, зеркало `ftd-detector` (тот ловит дно). Это **тактический** сигнал на горизонт **2–8 недель**, который предшествует коррекциям **10–20%** — не путать с `us-market-bubble-detector` (месяцы-годы, обвал 30%+).

## Зачем в плане

В **субботнем недельном блоке (Шаг 8)** даёт оценку риска вершины, которая питает вечерние прогоны всю неделю. Композит **≥ 41 (зона Orange)** — одно из двух условий включения **шорт-ветки (Шаг 6)** (второе — «≥ 3 distribution days за 25 сессий» из `ibd-distribution-day-monitor`). Достаточно одного из условий, чтобы система начала смотреть шорты.

## Терминология

- **Distribution day (день распределения)** — закрытие индекса вниз ≥ 0.2% на **большем** объёме, чем накануне = институционалы продают на силе.
- **Stalling day (день затухания)** — объём растёт, но рост цены < 0.1% (скрытое распределение); считается с весом 0.5.
- **Лидеры** — корзина растущих ETF как прокси силы рынка; их слом раньше индексов = ранний сигнал вершины.
- **Защитная ротация** — деньги уходят из роста (XLK/XLC/XLY/QQQ) в защиту (XLU/XLP/XLV/VNQ).

## Как работает

Композит = взвешенная сумма шести компонентов (каждый 0–100):

| # | Компонент | Вес | Что меряет |
| - | --------- | --- | ---------- |
| 1 | Distribution Day Count (O'Neil) | **25%** | Институциональные продажи за 25 сессий |
| 2 | Leading Stock Health (Minervini) | **20%** | Деградация корзины растущих ETF |
| 3 | Defensive Sector Rotation (Monty) | **15%** | Защита против роста (относительная сила) |
| 4 | Market Breadth Divergence | **15%** | Ширина (200MA авто + 50MA вручную) vs уровень индекса |
| 5 | Index Technical Condition | **15%** | Структура MA, проваленные отскоки, lower highs |
| 6 | Sentiment & Speculation | **10%** | VIX, put/call, term-структура |

Если оба коррелирующих компонента (distribution + defensive) ≥ 80, меньший по весу дисконтируется ×0.8 (чтобы не задваивать один и тот же сигнал).

### O'Neil — дни распределения (25%)

Считаются по S&P 500 и NASDAQ за 25 сессий, берётся **худший** индекс. Эффективный счёт = distribution + 0.5×stalling. Балл: **≥6 → 100**, 5 → 90, **4 → 75** (порог тревоги О'Нила), **3 → 55**, 2 → 30, 1 → 15, 0 → 0. Если ≥2 дней и ≥50% из них в последних 5 сессиях (кластеризация) — балл ×1.15.

### Minervini — деградация лидеров (20%)

Корзина по умолчанию: **ARKK, WCLD, IGV, XBI, SOXX, SMH, KWEB, TAN** (есть динамический режим — топ-N из пула по близости к 52-нед. максимуму). Каждый ETF набирает «балл деградации» 0–100: расстояние от 52-нед. максимума (−25%+ → 40, −15% → 30, −10% → 20, −5% → 10), ниже 50MA (+20) и 200MA (+20), паттерн lower-highs (+20). Итог = среднее; если ≥60% корзины «деградируют» (балл ≥50) — ×1.3.

### Monty — защитная ротация (15%)

Относительная доходность защиты минус рост, мультипериод **10д×0.2 + 20д×0.5 + 40д×0.3**. Балл по разнице: ≥+5% → 100, ≥+3% → 80, ≥+1.5% → 60, ≥+0.5% → 40, ≥0% → 20, отрицательная → 0 (рост ведёт = здорово). Если не все периоды защитные — потолок 80.

### Зоны риска (по композиту)

| Балл | Зона (цвет) | Бюджет риска | Действие |
| ---- | ----------- | ------------ | -------- |
| 0–20 | **Green** (норма) | 100% | Обычная работа |
| 21–40 | **Yellow** (раннее предупреждение) | 80–90% | Подтянуть стопы, меньше новых входов |
| **41–60** | **Orange** (повышенный риск) | 60–75% | Фиксация по слабым, без новых momentum-входов → **условие шорт-ветки** |
| 61–80 | **Red** (высокая вероятность вершины) | 40–55% | Агрессивная фиксация, хедж, шорт-лист |
| 81–100 | **Critical** (формирование вершины) | 20–35% | Максимальная защита, хедж, шорты по слабейшим |

Если данных по компоненту нет — его вес перераспределяется на доступные, в отчёте проставляется метка качества (Complete / Partial / Limited). При композите > 40 дополнительно проверяется FTD (для отслеживания возможного разворота вверх).

## Команда

Две цифры собираются вручную (через WebSearch) и передаются флагами; 200MA-ширина тянется автоматически из CSV TraderMonty.

```bash
python3 skills/market-top-detector/scripts/market_top_detector.py \
  --breadth-50dma 55 --breadth-50dma-date 2026-06-12 \
  --put-call 0.75 --put-call-date 2026-06-12 \
  --output-dir trading-data/market/
```

- `--breadth-50dma` — % акций S&P выше 50MA (диапазон 20–100).
- `--put-call` — equity put/call CBOE (диапазон 0.30–1.50).
- `--vix-term`, `--margin-debt-yoy` — опциональны (VIX-структура иначе определяется авто).

Выдаёт JSON + Markdown в `trading-data/market/`: композит и зона, сильнейший сигнал, разбивка по 6 компонентам, исторические аналоги вершин, what-if сценарии и delta к прошлому прогону.
