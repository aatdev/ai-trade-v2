# IBD Distribution Day Monitor — счётчик дней распределения

## Кратко

Считает по IBD-методике **дни распределения (distribution days)** для QQQ и SPY, ведёт их 25-сессионное истечение и 5% инвалидацию, оценивает класс риска **NORMAL / CAUTION / HIGH / SEVERE** и выдаёт рекомендацию по экспозиции TQQQ/QQQ. API-ключ не нужен — дневные бары QQQ/SPY идут через общий слой TradingView. Сделок не совершает.

> **Дистрибуция / институциональные продажи:** день распределения — это закрытие индекса **вниз ≥ 0.2% на бóльшем объёме**, чем накануне. Падение на растущем объёме = крупные игроки сбрасывают на силе. Накопление таких дней за короткое окно — классический IBD-признак ослабления аптренда. Парный к `ftd-detector` (тот ловит дно).

## Зачем в плане

В **субботнем недельном блоке (Шаг 8)** даёт счётчик дистрибуции, который питает вечерние прогоны всю неделю. **≥ 3 distribution days за 25 сессий** (класс CAUTION и выше) — одно из двух условий включения **шорт-ветки (Шаг 6)** (второе — «market-top ≥ 41 / Orange» из `market-top-detector`). Достаточно одного из условий.

## Терминология

- **Distribution day** — закрытие вниз ≥ 0.2% на объёме больше вчерашнего.
- **25-сессионное истечение** — DD удаляется из активного счёта, когда прошло > 25 торговых сессий.
- **5% инвалидация** — DD аннулируется, если индекс с момента DD вырос на 5% (по умолчанию по максимуму после DD; настраиваемо на close).
- **d5 / d15 / d25** — активные DD не старше 5 / 15 / 25 сессий (по возрасту 0..N включительно).
- **Кластер** — концентрация DD: d5 ≥ 2, d15 ≥ 3 или d25 ≥ 5.

## Как работает

### Детекция и жизненный цикл DD

DD фиксируется, когда **оба** условия: закрытие ≤ −0.2% к вчерашнему **и** объём > вчерашнего. Дальше DD ведётся, пока не выпадет по одной из причин (приоритет: invalidated > expired > active):

- **invalidated** — после DD был рост ≥ 5% (ищется хронологически первое пересечение порога в пределах окна истечения).
- **expired** — возраст > 25 сессий.

Сегодняшний DD не инвалидируется сразу (после него ещё нет сессий для оценки роста). Объём с отсутствующими/нулевыми значениями — это предупреждение в аудите, а не DD.

### Кластеры

Активные DD считаются в окнах: **d5** (≤5 сессий), **d15** (≤15), **d25** (≤25). Флаги кластера: `d5 ≥ 2`, `d15 ≥ 3`, `d25 ≥ 5` — ранняя концентрация продаж опаснее, чем те же дни, размазанные по 25 сессиям.

### Классы риска (на индекс)

| Класс | Условие |
| ----- | ------- |
| **NORMAL** | `d25 ≤ 2` |
| **CAUTION** | `d25 ≥ 3` |
| **HIGH** | `d25 ≥ 5` ИЛИ `d15 ≥ 3` ИЛИ `d5 ≥ 2` |
| **SEVERE** | `d25 ≥ 6` ИЛИ `d15 ≥ 4` ИЛИ (цена ниже 21EMA/50MA И `d25 ≥ 5`) |

Когда загружены оба индекса — применяется QQQ-взвешенная (TQQQ-aware) логика свода: любой SEVERE → SEVERE; QQQ HIGH → общий HIGH; QQQ NORMAL + SPY HIGH → HIGH (слабость широкого рынка перетекает в TQQQ); QQQ CAUTION + SPY CAUTION/HIGH → HIGH. Иначе — максимум из индексов.

### Рекомендация по экспозиции

TQQQ режется быстрее (3× дневное плечо, эффект ежедневного компаундинга на просадках):

| Класс | TQQQ → экспозиция / трейл-стоп | QQQ → экспозиция / трейл-стоп |
| ----- | ----------------------------- | ----------------------------- |
| NORMAL | 100% / base | 100% / base |
| CAUTION | 75% / min(base, 7%) — без новых добавок | 100% / min(base, 8%) |
| HIGH | 50% / min(base, 5%) | 75% / min(base, 6%) |
| SEVERE | 25% / min(base, 3%) — закрыть/хедж | 50% / min(base, 5%) |

Для TQQQ при HIGH/SEVERE предлагается альтернатива: частичный переход в QQQ или в кэш. Рекомендация — это управление риском, а не приказ на исполнение.

## Команда

```bash
python3 skills/ibd-distribution-day-monitor/scripts/ibd_monitor.py \
  --symbols QQQ,SPY \
  --instrument TQQQ \
  --current-exposure 100 \
  --base-trailing-stop 10 \
  --output-dir trading-data/market/
```

- `--symbols QQQ,SPY` — индексы для анализа (QQQ — прокси Nasdaq, SPY — S&P 500).
- `--lookback-days 80` — глубина истории (по умолчанию 80 сессий).
- `--as-of YYYY-MM-DD` — оценка на историческую сессию (бэктест).

Выдаёт JSON + Markdown в `trading-data/market/`: по каждому индексу — DD сегодня, активные/выбывшие дни, d5/d15/d25, флаги кластеров, фильтры 21EMA/50MA, класс риска; затем общий класс и рекомендация по экспозиции. Всегда указывается, какие даты дали активный счёт.
