# Trader Memory Core — память трейдера

Постоянный слой состояния, который связывает скрининг → анализ → сайзинг → управление позицией в единый **«тезис»** на каждую идею. Хранит, что вы думали, что произошло и какой вывод сделали — между сессиями.

Фаза 1 — одно-тикерные тезисы типов: `dividend_income`, `growth_momentum`, `mean_reversion`, `earnings_drift`, `pivot_breakout`.

Данные лежат в `trading-data/journal/theses/`: по одному `th_*.yaml` на тезис плюс `_index.json` (быстрый индекс). Целостность индекса гарантирует CLI — **не редактируйте YAML вручную**.

## Жизненный цикл тезиса

```
IDEA → ENTRY_READY → ACTIVE → (PARTIALLY_CLOSED) → CLOSED
                         └──────────────→ INVALIDATED
```

- **IDEA** — идея с экрана/скринера, входа ещё нет.
- **ENTRY_READY** — сетап готов, ждём триггер входа.
- **ACTIVE** — позиция открыта (есть фактическая цена и дата входа).
- **PARTIALLY_CLOSED** — часть позиции зафиксирована (trim).
- **CLOSED** — позиция закрыта, посчитан P&L, MAE/MFE, уроки.
- **INVALIDATED** — тезис отменён без полноценной сделки (или по kill-критерию).

Каждый переход пишется в `status_history` с причиной и временем. Поле `monitoring.next_review_date` отмечает дату следующего ревью; просроченные подсвечиваются в таблице (⚠).

## Операции (кнопки в UI)

Все операции выполняются через CLI `trader_memory_cli.py` в фоне; прогресс виден в живом логе. Деструктивные (удаление, закрытие) требуют подтверждения.

### Ingest — зарегистрировать тезис

Создаёт тезис(ы) из JSON-вывода скринера.

- **Источник** — имя скила-источника (например `vcp-screener`, `kanchi-dividend-sop`, `earnings-trade-analyzer`).
- **Файл** — относительный путь к JSON-отчёту (например `reports/vcp_2026-06-12.json`). Абсолютные пути и `..` запрещены.

Дубликаты отсекаются по `origin_fingerprint`.

### Store — операции над состоянием

- **Сменить статус (transition)** — перевести тезис в новый статус с обязательной причиной; опц. `--event-date` (бэкдейт записи истории).
- **Открыть позицию (open-position)** — зафиксировать фактическую цену и дату входа (→ ACTIVE); опц. число акций, причина, дата события.
- **Прикрепить сайзинг (attach-position)** — присоединить JSON-отчёт position-sizer (относительный путь); опц. ожидаемые вход/стоп.
- **Частично закрыть (trim)** — продать часть позиции: количество, цена, дата (→ PARTIALLY_CLOSED, либо CLOSED при полном выходе); опц. причина и причина выхода.
- **Закрыть (close)** — закрыть ACTIVE-тезис: причина выхода (`stop_hit` / `target_hit` / `time_stop` / `invalidated` / `manual`), цена и дата выхода. Считает реализованный P&L.
- **Завершить (terminate)** — перевести в терминальный статус CLOSED/INVALIDATED (например, отмена идеи без сделки); опц. цена/дата выхода.
- **Отметить ревью (mark-reviewed)** — записать ревью с оценкой OK/WARN/REVIEW и заметкой; сдвигает `next_review_date` на интервал.
- **Список / Показать (list / get)** — отфильтрованный список тезисов (тикер/статус/тип/даты) и вывод одного тезиса в YAML.
- **Перестроить индекс (rebuild-index)** / **Doctor** — служебные: пересборка `_index.json` из YAML и проверка консистентности файлов и индекса.

### Heat — портфельный риск

- **Portfolio heat** — открытый риск по ACTIVE-тезисам (для `breakout-trade-planner`); опц. капитал, лимит heat %, макс. позиций, режим «только JSON».

### Review — ревью и постмортемы

- **Просроченные ревью (review-due)** — список тезисов, у которых `next_review_date <= сегодня`.
- **Сводка (summary)** — агрегированная статистика по тезисам.
- **Постмортем (postmortem)** — отчёт по закрытому тезису с P&L и MAE/MFE (максимальная просадка / максимальный профит в ходе сделки). MAE/MFE берутся из общего слоя данных TradingView (без API-ключа); если мост недоступен — метрики деградируют до `null`, постмортем всё равно строится.

### Delete — удалить тезис

Жёсткое удаление: убирает `th_*.yaml` и запись из индекса. Для штатного выхода из сделки используйте **Закрыть/Завершить**, а не удаление — удаляйте только мусорные/ошибочные записи.

## Ключевые поля тезиса

- **thesis_statement** — формулировка идеи (что и почему).
- **entry** — `target_price`, условия входа, фактические цена/дата.
- **exit** — `stop_loss`, `take_profit`, `take_profit_rr`, `time_stop_days`, причина и факт выхода.
- **position** — число акций, объём, риск в долларах, метод сайзинга.
- **monitoring** — интервал и дата ревью, статус ревью, триггеры, алерты.
- **outcome** — `pnl_dollars` / `pnl_pct`, дни в позиции, MAE/MFE, выводы (`lessons_learned`).
- **origin** / **linked_reports** — источник (скил, файл, грейд) и связанные отчёты.
- **status_history** — таймлайн статусов с причинами.

## Как это собрано в конвейер

```
Скринер → ingest (IDEA) → анализ/валидация → transition (ENTRY_READY → ACTIVE)
       → сайзинг (attach/open-position) → исполнение → периодические review
       → close + postmortem (журнал с MAE/MFE)
```

Полное описание команд и флагов — в `SKILL.md` скила и `references/thesis_lifecycle.md` (на английском).
