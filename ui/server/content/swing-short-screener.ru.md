# Swing Short Screener — поиск шорт-кандидатов (Stage 4)

## Кратко

Сканирует S&P 500 (или кастомную вселенную) на **Stage 4 слабость** — зеркало vcp-screener для шорта: там Stage 2 сила у пробоя вверх, здесь Stage 4 слабость на пробое вниз. По каждому имени считается 5-факторная weighted-модель слабости (0–100), присваивается грейд **A/B/C/D**, выдаётся JSON + Markdown watchlist с уровнями вход/стоп/2R. **Detection-only — ордера не шлёт.** API-ключ не нужен (дневные OHLCV через общий слой TradingView); есть офлайн `--fixture` для тестов.

> Кандидат проходит жёсткую инвалидацию ПЕРЕД скорингом, и только выжившие оцениваются по 5 факторам.

## Зачем в плане

**Шаг 6.1 — шорт-кандидаты.** Когда шорт-ветка разрешена (слабый рынок, нет свежего FTD, оборонительная экспозиция), этот скринер даёт graded-список имён, слабых относительно индекса и ломающих базу. Уровни вход/стоп/2R уходят дальше в planner шорта; перед входом человек подтверждает borrow/SSR у брокера.

## Терминология

- **Stage 4** — фаза устойчивого падения (по Вайнштайну/Минервини): цена ниже ключевых средних, средние выровнены медвежьи. Кандидат для шорта.
- **Lower highs** — нисходящая структура свинг-максимумов: каждый локальный максимум ниже предыдущего = подтверждение нисходящего тренда.
- **Borrow / HTB** — наличие бумаги «в долг» под шорт; HTB (hard-to-borrow) — труднозаимствуемая бумага, высокий риск принудительного выкупа / сквиза. Скрипт это **не** проверяет — напоминает подтвердить у брокера.
- **Death cross** — MA50 ниже MA200.

## Как работает

### Жёсткая инвалидация (отсев до скоринга)

Кандидат отбрасывается, если выполнено любое: история < 200 сессий (нельзя посчитать MA200); цена **выше** MA200 (не Stage 4); цена < `--min-price` (по умолчанию $5); средний дневной оборот < `--min-dollar-vol` (по умолчанию $3M, риск locate/сквиза).

### 5-факторная weighted-модель (из кода)

| Фактор | Вес | Как считается |
|---|---|---|
| **Trend Structure** | 30% | аддитивно: ниже MA200 → +40, death cross → +25, ниже MA50 → +20, MA50 падает → +15 |
| **Relative Strength** | 25% | `rel = доходность − доходность SPY` за lookback (63 сессии): rel ≤ −20% → 100, rel ≥ 0 → 0, линейно между |
| **Base Breakdown** | 20% | пробой 20-дн. поддержки → +50; расширение объёма `clamp((vol_ratio−1)×50, 0, 50)` (всплеск 2x добавляет полные 50) |
| **Lower Highs** | 15% | `pct = (прежний 20-дн. максимум − текущий)/прежний`; −10% и ниже → 100 |
| **Liquidity / Borrow** | 10% | оборот ≥ $50M → 100, ≥ $10M → 60, ≥ $3M → 30, иначе 10; цена < $5 делит балл пополам |

`composite = Σ(фактор × вес)`. Чем выше — тем подтверждённее Stage 4 слабость.

### Грейды

| Score | Грейд | Трактовка |
|---|---|---|
| 80–100 | **A** | Чистая Stage 4 слабость — приоритетный шорт |
| 65–79 | **B** | Сильная слабость — торговать на подтверждённом пробое |
| 50–64 | **C** | Развивающаяся — watchlist, ждать чистого пробоя/ретеста |
| < 50 | **D** | Слабый сигнал — пропуск |

**State cap:** если RSI(14) < 25 ИЛИ цена > 20% ниже MA50 — это «падающий нож» (риск отскока), грейд капается до **C** (метка ★). Сырой `raw_grade` сохраняется в JSON.

### Уровни сделки (по кандидату)

- **Вход** — текущая цена (уровень пробоя).
- **Стоп** — НАД последним нижним максимумом за 20 сессий + буфер 0.5×ATR (fallback — 20-дн. максимум).
- **2R-цель** — `вход − 2 × (стоп − вход)`.
- **Стоп-дистанция 2–10%:** кандидаты со стопом вне коридора отбрасываются (`--min-stop-pct`/`--max-stop-pct`) — ниже пол сидит в шуме, выше потолок это пост-кризисный мусор.

## Команда

Команда из плана (Шаг 6.1, шорт-кандидаты):

```bash
python3 skills/swing-short-screener/scripts/screen_short.py \
  --full-sp500 \
  --min-grade B \
  --top 10
```

`--output-dir` по умолчанию указывает в `trading-data/screeners/` (`$TRADING_DATE_DIR/screeners`, иначе `reports/`). Выдаёт `swing_short_screener_<date>[_<time>].json` / `.md` (`meta` + `candidates[]` с компонентами, грейдом, метриками и уровнями).
