# FTD Detector — подтверждение дна рынка

## Кратко

Ищет **Follow-Through Day (FTD)** — сигнал Уильяма О'Нила, подтверждающий, что коррекция закончилась и начался новый рост. Работает как «наступательный» индикатор: говорит, **когда можно снова наращивать лонг-экспозицию** после падения. Зеркало `market-top-detector` (тот, наоборот, ловит вершину). API-ключ не нужен — данные по S&P 500 и NASDAQ идут через общий слой TradingView.

> Главное правило О'Нила: без FTD ни один устойчивый разворот вверх исторически не начинался. Но FTD — условие **необходимое, но не достаточное**: ~75% сигналов проваливаются, поэтому есть оценка качества (0–100) и мониторинг «здоровья» после FTD.

## Зачем в плане

В **субботнем недельном блоке (Шаг 8)** запускается после коррекций ≥3%, чтобы понять: рынок развернулся или ещё падает. Результат напрямую влияет на **шорт-ветку (Шаг 6)**: одно из трёх условий включения шортов — «**нет свежего подтверждённого FTD**». То есть если FTD подтверждён — шорты запрещены, система смотрит только лонг.

## Терминология

- **Коррекция** — падение закрытия индекса ≥3% от недавнего максимума и ≥3 «падающих» дня по пути вниз.
- **Swing low** — самое низкое **закрытие** во время коррекции; точка отсчёта для отскока. Это локальный минимум.
- **Rally attempt (попытка отскока)** — дни 1–3 от swing low. Это ещё **не** сигнал к покупке.
- **Day 1** — первый день отскока: закрытие выше предыдущего **или** в верхних 50% дневного диапазона.
- **FTD window** — дни 4–10 отскока, окно, в котором ищется сам FTD.
- **Distribution day (день распределения)** — падение ≥0.2% на **возросшем** объёме = институционалы продают. После FTD это плохой признак.
- **Power Trend** — подтверждение силы тренда: 21EMA > 50SMA, 50SMA растёт, цена выше 21EMA.
- **Invalidation (отмена)** — закрытие ниже минимума (low) дня FTD обнуляет сигнал.
- **Dual-index** — сигнал отслеживается сразу по двум индексам: S&P 500 (`^GSPC`) и NASDAQ (через `QQQ`).

## Как работает

### Конечный автомат

```
NO_SIGNAL → CORRECTION → RALLY_ATTEMPT → FTD_WINDOW → FTD_CONFIRMED
                ↑              ↓               ↓              ↓
                └── RALLY_FAILED ←────────────┘       FTD_INVALIDATED
```

Анализ идёт по каждому индексу отдельно, окно — последние 60 торговых дней.

1. **Поиск swing low.** Падение от недавнего максимума (в пределах 40 дней) ≤ −3%, ≥3 падающих дня, день — локальный минимум по закрытию.
2. **Отслеживание отскока.** От swing low ищется Day 1 (см. терминологию), дальше считаются Day 2, 3… Отскок отменяется, если: любое закрытие ниже swing low, либо на Day 2–3 закрытие ниже минимума Day 1.
3. **Поиск FTD (дни 4–10).** Нужны **все** условия: рост ≥ **1.25%** и объём **строго больше** предыдущего дня. Тиры роста: ≥2.0% (strong) / ≥1.5% (recommended) / ≥1.25% (minimum). Бонусом сравнивается объём со средним за 50 дней (без заглядывания в будущее).
4. **Объединение двух индексов.** FTD хотя бы на одном индексе → сигнал есть; FTD на обоих → «двойное подтверждение» (+15 к качеству).

### Здоровье после FTD

Запускается только при подтверждённом FTD:

- **Дни распределения** в первые 5 дней: чем раньше — тем хуже (День 1–2 → −30, День 3 → −15, День 4–5 → −5; ≥2 дней → ещё −20).
- **Инвалидация:** закрытие ниже low дня FTD → **score обнуляется**, сигнал `Failed/Invalidated`.
- **Power Trend:** проверка по S&P 500 (нужно 50+ дней).

### Оценка качества (0–100)

| Балл | Сигнал | Рекомендуемая лонг-экспозиция |
| ---- | ------ | ----------------------------- |
| 80–100 | Strong FTD | 75–100% |
| 60–79 | Moderate FTD | 50–75% |
| 40–59 | Weak FTD | 25–50% |
| < 40 | No FTD / Failed | 0–25% |

Баллы складываются из: база за день FTD (Day 4–7 = 60, Day 8–10 = 50), бонус за рост (+5/+10/+15), объём выше среднего (+10), двойное подтверждение (+15), здоровье после FTD (±). Инвалидация перекрывает всё.

## Команда

```bash
python3 skills/ftd-detector/scripts/ftd_detector.py
```

Выдаёт JSON + Markdown в `trading-data/market/`. Внутри: объединённое состояние, какой индекс дал сигнал, swing low / rally / FTD по каждому индексу, итоговый score с разбивкой, дни распределения, инвалидация, Power Trend.
