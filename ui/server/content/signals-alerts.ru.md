# Signals Alerts — синхронизация алертов TradingView с журналом сигналов

## Кратко

Создаёт, удаляет и **синхронизирует алерты в TradingView Desktop** (через CDP) по журналу `trading-data/analysis/signals.md`. По каждому сигналу ставит **5 алертов приоритетного сценария**: Trigger / Stop / T1 / T2 / T3. Sync работает как **diff**, а не «снести всё и пересоздать»: что совпало по тексту — не трогает, чего нет — создаёт, устаревшее — удаляет. Нужен запущенный TradingView Desktop.

> Тяжёлую работу делают Node-скрипты через единое CDP-соединение (один запуск = одна сессия, без оверхеда на десятки MCP-вызовов). Скил парсит запрос, выбирает скрипт, форматирует отчёт.

## Зачем в плане

**Шаг 5.5** (лонг) и **Шаг 6.5** (шорт): после того как построен план и идея записана в журнал, алерты ставятся одним sync-вызовом. Автоматические алерты watchlist помечаются тегом **`[WL]`**. Устаревшие `[WL]`-алерты система снимает сама (например, по сбежавшим тикерам в `intraday`), а **ручные алерты журнала не трогаются** — удаление точечное по подстроке в сообщении.

## Терминология

- **CDP (Chrome DevTools Protocol)** — канал управления TradingView Desktop на `:9222`; через него скрипты ставят/снимают алерты и сохраняют layout, как будто кликает человек.
- **Trigger** — алерт на вход: пересечение цены триггера (LONG — вверх, SHORT — вниз), опционально с объёмным условием.
- **Stop** — алерт на закрытие по стопу (LONG — вниз, SHORT — вверх).
- **T1 / T2 / T3** — алерты на цели фиксации (T2/T3 опциональны).
- **Diff-sync** — приведение набора алертов к плану минимумом изменений (ключ совпадения — текст `message`).
- **`[WL]`** — тег авто-алертов watchlist; отделяет «системные» алерты от ручных при удалении.

## Как работает

### Источник и план

`parse_signals.mjs` читает `signals.md`, по каждому блоку берёт **приоритетный сценарий** (направление + Trigger / Stop / T1 / T2 / T3) и строит JSON-план с готовыми 5 алертами на сигнал. Для **SHORT** условия зеркальны: Trigger/T1/T2/T3 — `Crossing Down`, Stop — `Crossing Up`. Если в Trigger-строке указан объёмный фильтр («на объёме > 40M»), парсер вытащит значение и создаст **один** алерт с двумя условиями (цена + объём через «Add condition»). Сигнал без обязательных полей (Trigger/Stop/T1) уходит в `skipped`.

### Действия

| Действие | Что делает |
| --- | --- |
| `list` (дефолт) | Показывает план таблицей, ничего не меняет в TradingView. |
| `create` | Создаёт недостающие алерты (дедуп по `message`), сохраняет layout. |
| `delete` | Точечно сносит «наши» алерты по тикерам (purge или diff). |
| `sync` | Diff: сначала удаляет устаревшие, потом досоздаёт недостающие. |

### Sync (diff, два шага по порядку)

1. **Удалить только устаревшие** «наши» алерты, которых уже нет в новом плане (`delete_alerts.mjs --keep-from-plan`); совпавшие остаются жить со своей историей.
2. **Создать недостающие** (`create_alerts.mjs`); дедуп по `message` пропустит уже существующие.

Если уровень в `signals.md` изменился — новый `message` содержит новую цену, старый алерт попадает в «устаревшие» и пересоздаётся. Алерты по тикерам не из плана не затрагиваются. После создания/снятия маркер-линий скрипты **сохраняют layout** графика (иначе линии вернутся при следующей загрузке).

### Жёсткие правила

- Никогда не вызывать `delete_all=true` — это снесёт ВСЕ алерты пользователя, включая чужие. Удаление только точечное.
- Скрипты не запускать параллельно (каждый держит свой CDP-сеанс, конкуренция ломает UI-диалоги).
- Скил только **читает** `signals.md`, никогда не пишет в него.
- Перед create/delete/sync — `tv_health_check`; если TradingView Desktop недоступен, скил останавливается и просит запустить `tv launch`.

## Команда

Плановый вызов (Шаг 5.5 / 6.5) — headless `claude -p`:

```bash
claude -p "Запусти skill signals-alerts в режиме sync: синхронизируй алерты \
TradingView с журналом trading-data/analysis/signals.md" \
  --permission-mode bypassPermissions --output-format text
```

Под капотом (если нужно дёрнуть скрипты напрямую — например, снять только `[WL]` по сбежавшему тикеру):

```bash
# план в JSON
node .claude/skills/signals-alerts/scripts/parse_signals.mjs [--tickers BSX,LULU]

# sync: удалить устаревшие → создать недостающие
node .claude/skills/signals-alerts/scripts/delete_alerts.mjs --keep-from-plan --file plan.json
cat plan.json | node .claude/skills/signals-alerts/scripts/create_alerts.mjs

# снять только авто-алерты watchlist по тикеру (ручные не трогаются)
node .claude/skills/signals-alerts/scripts/delete_alerts.mjs --tickers NVDA --message-contains "[WL]"
```

Источник сигналов — `trading-data/analysis/signals.md` (читается). Результат — изменения прямо в TradingView Desktop; временные планы кладутся в `./tmp/`, в `reports/` ничего не пишется.
