# Видение проекта Claude Trading Skills

Версия: 0.1
Последнее обновление: 2026-05-17

Английская версия: [PROJECT_VISION.md](PROJECT_VISION.md)
Японская версия: [PROJECT_VISION.ja.md](PROJECT_VISION.ja.md)

Claude Trading Skills — это операционная система торгового процесса на базе Claude Skills для частных инвесторов с ограниченным временем.

Она помогает инвесторам использовать долгосрочное инвестирование как фундамент, дисциплинированно брать возможности для свинг-трейдинга только тогда, когда это позволяют рыночные условия, и непрерывно совершенствоваться через управление рисками, ведение журнала и разбор сделок.

## 1. Мантра

**Empower solo traders, growing together.**
(«Расширяем возможности трейдеров-одиночек, растём вместе».)

Этот проект существует, чтобы помочь частным трейдерам выйти за рамки изолированных дискреционных решений и выстроить с помощью Claude Skills воспроизводимые процессы, явное управление рисками, ведение журнала, разбор сделок и циклы непрерывного улучшения.

Здесь **solo** (в одиночку) не означает изоляцию. Проект рассчитан на трейдеров, которые принимают собственные решения и берут на себя собственный риск, но при этом позволяют рабочим процессам, разборам и практикам улучшения развиваться через совместную практику.

## 2. Важное уведомление и отказ от ответственности

Этот репозиторий предоставляет Claude Skills и сопутствующие материалы в образовательных, исследовательских целях и для улучшения процессов. Он не предоставляет финансовых консультаций, инвестиционно-консультационных услуг, торговых сигналов, исполнения сделок у брокера, налоговых или юридических консультаций.

Инвестирование и трейдинг сопряжены с риском, включая потерю основного капитала. Исторические данные, бэктесты, результаты скрининга, образцы отчётов и сгенерированный ИИ анализ не гарантируют будущих результатов. Все торговые решения, определение размера позиции, управление рисками, соблюдение налоговых/регуляторных требований и решения об использовании брокера являются исключительной ответственностью пользователя.

Этот проект распространяется по лицензии MIT. Программное обеспечение и материалы предоставляются **«КАК ЕСТЬ, БЕЗ КАКИХ-ЛИБО ГАРАНТИЙ»**, как указано в лицензии.

## 3. Происхождение и почему это open source

Этот проект начался потому, что автор хотел использовать ИИ для улучшения собственного торгового процесса.

Для частных трейдеров — особенно тех, кто инвестирует и торгует, совмещая это с работой, семьёй и другими жизненными обязанностями, — время, информационная перегрузка, контроль эмоций и управление рисками являются реальными ограничениями. Claude Trading Skills был создан, чтобы упростить поддержание процесса обзора рынка, генерации кандидатов, планирования сделок, управления рисками, ведения журнала и разбора сделок в этих условиях.

Проект прежде всего является практическим инструментом, который автор использует ежедневно и еженедельно. Он существует, чтобы улучшить собственный торговый процесс автора, выстроить более устойчивый процесс принятия решений и продолжать учиться на записях и разборах. В то же время он является open source, потому что те же рабочие процессы могут быть полезны другим людям, сталкивающимся с похожими ограничениями.

Позиция проекта — **сначала для себя, открыто для других**. Первый пользователь — это автор. Даже когда внешней обратной связи мало, проект должен продолжать улучшаться как инструмент, которым автор реально пользуется. Сверх этого проект стремится позволить рабочим процессам, разборам и практикам улучшения накапливаться как общее практическое знание для частных трейдеров.

Claude Trading Skills не предлагает готовой формулы выигрыша. Автор сам — трейдер, который продолжает учиться. Этот репозиторий — не место для раздачи «ответов». Это место для обмена практическими рабочими процессами, которые помогают частным трейдерам укреплять собственное суждение, управлять рисками, учиться на записях и постепенно совершенствоваться.

## 4. Основной тезис

Цель этого проекта — не предсказывать рынки лучше всех остальных.

Цель — помочь частным инвесторам и трейдерам принимать решения, которые более структурированы, осознанны в отношении рисков, поддаются разбору и улучшению.

Основной поток этого проекта — не:

```text
Вопрос -> Сигнал -> Сделка
```

Основной цикл — это:

```text
План -> Сделка -> Запись -> Разбор -> Улучшение
```

Claude Trading Skills — это не движок сигналов. Он стремится быть **операционной системой процесса принятия решений**, которая помогает частным трейдерам использовать ИИ для построения более устойчивого процесса принятия решений, последовательно управляя суждением, риском, записями и улучшением.

## 5. Назначение

Claude Trading Skills — это репозиторий Claude Skills, сфокусированный на акциях США, ETF, дивидендном инвестировании, свинг-трейдинге, макроанализе и исследовании стратегий, с включением опционных и событийных стратегий там, где это уместно.

Скилы поддерживают такие решения, как:

- Понимание рыночных условий
- Скрининг кандидатов для торговли или инвестиций
- Построение торговых планов
- Расчёт размера позиции
- Обзор риска портфеля
- Запись торговых гипотез
- Разбор результатов и выявление улучшений
- Тестирование новых идей стратегий

Проект — это не просто набор полезных аналитических инструментов. Он стремится быть **системой поддержки принятия решений** для частных трейдеров.

В частности, он стремится помочь частным инвесторам, которые используют долгосрочное инвестирование как базу и хотят эффективно использовать ограниченное время для свинг-трейдинга, когда это позволяют рыночные условия, не принимая чрезмерного риска и совершенствуясь через записи и разбор.

## 6. Чем этот проект является / не является

| Этот проект — это | Этот проект — это не |
| --- | --- |
| Система поддержки принятия решений | Финансовая консультация или инвестиционно-консультационная услуга |
| Набор инструментов для торгового процесса | Сервис сигналов на покупку/продажу |
| Фреймворк управления рисками и разбора | Система гарантии прибыли |
| Репозиторий Claude Skills | Платформа исполнения сделок у брокера |
| Система, поддерживающая обучающий цикл трейдера | Полностью автоматизированный торговый бот |

Этот проект не заменяет суждение трейдера. Он помогает сделать процесс принятия решений трейдера явным, воспроизводимым, поддающимся разбору и улучшению.

Окончательные решения и ответственность за риск всегда остаются за пользователем. Подробный отказ от ответственности централизован в разделе «Важное уведомление и отказ от ответственности», и эта позиция не должна меняться по мере роста проекта.

## 7. Целевые пользователи

Основная аудитория — **частные инвесторы и свинг-трейдеры с ограниченным временем**.

Более конкретно, проект рассчитан на людей, которые используют долгосрочное и дивидендное инвестирование как фундамент для накопления капитала, стремясь к дополнительной доходности через кратко- и среднесрочные свинг-сделки, когда рыночные условия благоприятны. У них есть ограничения, связанные с работой, семьёй или жизнью, поэтому им нужно делать ежедневные решения более эффективными и систематизировать управление рисками и ведение журнала.

### Основные пользователи

Первые пользователи, которым отдаётся приоритет:

| Персона | Главная цель | Необходимый путь |
| --- | --- | --- |
| Свинг-трейдер на неполную занятость | Ежедневный обзор рынка, генерация кандидатов, планирование сделок | `market-regime-daily` / `swing-opportunity-daily` |
| Растущий инвестор, желающий более сильного контроля рисков | Разделять наступательные и оборонительные рынки | рабочий процесс `market-regime` / `exposure` |
| Дивидендный и долгосрочный инвестор | Находить дивидендные акции, проверять активы, оценивать структуру портфеля | `core-portfolio` / `dividend-income` |

### Вторичные / продвинутые пользователи

Проект также может поддерживать:

| Персона | Главная цель | Необходимый путь |
| --- | --- | --- |
| Событийный / отчётный трейдер | Находить возможности после отчётов, новостей или экономических событий | `advanced-satellite` / `earnings-event` |
| Трейдер по коротким стратегиям | Отслеживать слабые или перегретые акции в risk-off среде | `advanced-satellite` / `risk-off-short` |
| Исследователь / разработчик стратегий | Тестировать гипотезы и улучшать стратегии | `strategy-research` |
| Продвинутый пользователь | Расширять кастомные рабочие процессы, YAML-манифесты и CLI-инструменты | манифесты / скрипты / матрица API |

Этот проект изначально не рассчитан на:

- Пользователей, ожидающих полностью автоматизированной торговли
- Пользователей, которым нужна гарантированная прибыль или отданные на аутсорс торговые сигналы
- Пользователей, которые не хотят управлять рисками или вести записи
- Пользователей, сфокусированных только на краткосрочном скальпинге

## 8. Операционная философия Core + Satellite

Основной пользователь этого проекта работает по структуре **Core + Satellite** (ядро + спутник):

- **Core (ядро)**: Долгосрочное инвестирование, дивидендные акции, ETF, управление портфелем
- **Satellite (спутник)**: Свинг-трейдинг, темы, пробои, постотчётный моментум
- **Advanced Satellite (продвинутый спутник)**: Короткие стратегии, событийные стратегии, опционные стратегии там, где это уместно
- **Shared Layer (общий слой)**: Рыночный режим, управление рисками, определение размера позиции, ведение журнала, разбор

Ключевой момент — не смешивать назначение, таймфрейм и профиль риска активностей Core и Satellite.

Этот проект предоставляет операционный процесс для частных инвесторов, которые используют долгосрочное накопление капитала как фундамент и стремятся к дисциплинированным кратко- и среднесрочным торговым возможностям только тогда, когда это позволяют рыночные условия.

Advanced Satellite включает области с более высоким риском, более сложными предпосылками или дополнительными ограничениями исполнения. Короткие стратегии, событийные стратегии и опционные стратегии следует использовать только при достаточном опыте, явных лимитах убытка, ручном контроле, понимании ограничений API и брокера, а также предварительно проверенных рабочих процессах. Тщательно реализованный скил не обязательно является частью первого рекомендуемого пути для основного пользователя.

## 9. Текущее состояние

Этот репозиторий уже содержит множество скилов. В широком смысле они охватывают:

- **Анализ рынка**: Рыночный режим, breadth, секторы, макро, новости, риск пузыря
- **Скрининг**: CANSLIM, VCP, дивидендные акции, постотчётный моментум, темы, институциональные потоки
- **Планирование сделок**: Пробои, Parabolic Short, определение размера позиции, управление экспозицией
- **Портфель и записи**: Управление портфелем, отслеживание гипотез, записи сделок, разборы (postmortem)
- **Исследование стратегий**: Бэктестинг, генерация идей стратегий, конвейер исследования эджа (edge)
- **Качество и мета-скилы**: Дизайн скилов, ревью, интеграционное тестирование, циклы улучшения

Отдельные скилы стали существенными. С точки зрения пользователя остаётся несколько проблем:

- Трудно понять, какой скил использовать
- Трудно увидеть, как должны выстраиваться в последовательность несколько скилов
- Полный рабочий процесс до и после сделки ещё не упакован чётко
- GitHub и `.skill`-файлы могут пугать менее технических пользователей
- Выводы скилов ещё не полностью связаны с непрерывным обучающим циклом

Следующий этап — это не столько добавление новых скилов, сколько **объединение скилов в пригодные для использования формы**, **направление пользователей на правильный путь** и **превращение их в операционные рабочие процессы**.

## 10. Стратегическое направление: от коллекции скилов к Trading OS

Следующее направление проекта можно сформулировать так:

> От коллекции скилов к Trading OS для трейдеров-одиночек.

Под Trading OS мы понимаем слой рабочих процессов, который связывает рыночный контекст, бюджетирование риска, выбор стратегии, генерацию кандидатов, планирование сделок, ручные шлюзы исполнения, ведение журнала, разбор и улучшение для ежедневных и еженедельных инвестиционных решений.

Эта Trading OS не предназначена для того, чтобы дать частным трейдерам лёгкий выигрыш. Она предназначена помочь им оставаться в игре, управлять рисками, учиться на записях и постепенно улучшать процесс принятия решений даже при ограниченном времени.

Идеальный опыт выглядит так:

1. Пользователь на естественном языке описывает, какой именно торговлей или инвестированием он хочет заниматься
2. Система понимает его цели, опыт, бюджет времени, толерантность к риску и API-окружение
3. Она рекомендует подходящий набор скилов и рабочий процесс
4. Пользователь может по порядку проверить рыночные условия, риск, кандидатов и торговые планы
5. После сделки пользователь записывает гипотезу и результат, затем передаёт уроки в следующую итерацию

Если этот поток работает, то даже пользователи, которым некомфортно с GitHub или настройкой инструментов, могут шаг за шагом получить доступ к ценности проекта.

## 11. Архитектура проекта

Проект следует организовать по этим слоям:

```text
1. Отдельные скилы (Individual Skills)
    |
2. Инвентарь скилов / Матрица API (Skill Inventory / API Matrix)
    |
3. Наборы скилов (Skillsets)
    |
4. Рабочие процессы (Workflows)
    |
5. Trading Skills Navigator
    |
6. Точки входа пользователя (User Entry Points)
    |
7. Журнал / Разбор / Обучающий цикл (Journal / Postmortem / Learning Loop)
```

Каждый слой имеет отдельную роль:

| Слой | Роль |
| --- | --- |
| Скилы | Небольшие специализированные возможности для анализа, расчёта, планирования и записей |
| Инвентарь скилов / Матрица API | Единый источник истины по сценариям использования, требуемым API, сложности и входам/выходам |
| Наборы скилов | Целевые бандлы, которые определяют, какие скилы относятся друг к другу |
| Рабочие процессы | Операционные последовательности, шлюзы решений и передачи артефактов |
| Navigator | Интерактивный гид, который рекомендует наборы скилов и рабочие процессы на основе целей пользователя |
| Точки входа пользователя | Документация, стартовые промпты, CLI, будущий веб-интерфейс и другие способы начать пользоваться проектом |
| Обучающий цикл | Механизмы ведения журнала, разбора и улучшения, которые растят и скилы, и трейдера |

Эта структура снижает рассинхронизацию между исходными файлами скилов, документацией, логикой рекомендаций и рабочими процессами. Navigator предоставляет интерактивные рекомендации, тогда как Точки входа пользователя предоставляют статическую документацию, стартовые промпты, CLI-инструменты и будущие пути веб-интерфейса.

## 13. Метрики успеха

## 14. Принципы дизайна

Будущая разработка должна следовать этим принципам:

1. **Процесс важнее прогноза**
   - Отдавать приоритет воспроизводимым процессам принятия решений, а не самим прогнозам.

2. **Сначала риск**
   - Анализировать рыночные условия и бюджет риска до кандидатов или сигналов.

3. **Человеческое суждение остаётся центральным**
   - Проект поддерживает человеческое суждение; он не автоматизирует торговые решения.

4. **Рекомендации должны быть объяснимы**
   - Система должна объяснять, почему предлагается тот или иной скил, рабочий процесс или настройка риска.

5. **Компонуй небольшие скилы**
   - Предпочитать небольшие сфокусированные скилы, скомпонованные вместе, большим универсальным скилам.

6. **Доступно для новичков, расширяемо для продвинутых пользователей**
   - Предоставлять простые точки входа для пользователей без GitHub, сохраняя место для глубокой кастомизации.

7. **Записывай и улучшай**
   - Сделка не заканчивается исполнением; она заканчивается, когда записана, разобрана и использована для улучшения.

8. **Поддерживай единый источник истины**
   - Метаданные скилов должны жить в как можно меньшем числе канонических мест. Каталоги, матрицы API, рекомендации Navigator и манифесты рабочих процессов должны по возможности генерироваться из общих метаданных или валидироваться против них.

## 17. Долгосрочное видение

В долгосрочной перспективе этот проект должен стать местом, где:

- Частные трейдеры могут найти стратегии и рабочие процессы, которые им подходят
- Рыночный режим может направлять экспозицию к риску
- Торговые планы явны до исполнения
- Результаты записываются после сделок и используются для обучения
- Скилы и рабочие процессы улучшаются через практику
- У новичков есть доступные точки входа, а у продвинутых пользователей — место для расширения

В конечном итоге пользователи должны иметь возможность делиться рецептами рабочих процессов, шаблонами разборов, заметками по исследованию стратегий и предложениями по улучшению скилов, чтобы практическое знание частных трейдеров могло улучшать весь проект.

Суть этого проекта — не вручить трейдерам «ответ».

Она в том, чтобы создать среду, где трейдеры могут укреплять своё суждение, управлять рисками, учиться на записях и непрерывно расти.
